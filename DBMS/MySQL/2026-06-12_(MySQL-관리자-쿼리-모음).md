## 1. 핵심 요약
- MySQL 8.0 User 생성 시 인증방식(`mysql_native_password` vs `caching_sha2_password`), 비밀번호 정책, 계정 잠금 옵션을 명시하는 것이 권장됨.
- 권한 분류: Administrative(전역) / Database / Object 권한으로 나뉘며, Dynamic 권한은 플러그인 설치 후 사용 가능.
- InnoDB History Length 증가: 장기 실행 트랜잭션이 원인. `SHOW ENGINE INNODB STATUS`의 History list length 주목.
- Audit Plugin(Percona): `audit_log_exclude_accounts`와 `audit_log_include_accounts`는 동시에 값 설정 불가.

## 2. 상세 설명

### User 생성 및 권한 관리

**User 생성 (전체 옵션 명시)**:
```sql
CREATE USER 'user'@'%'
   IDENTIFIED WITH 'mysql_native_password' BY 'password'
   REQUIRE NONE
   PASSWORD EXPIRE INTERVAL 30 DAY       -- 30일 후 만료
   ACCOUNT UNLOCK
   PASSWORD HISTORY DEFAULT              -- 기존 비밀번호 재사용 금지
   PASSWORD REUSE INTERVAL DEFAULT
   PASSWORD REQUIRE CURRENT DEFAULT;

-- 옵션 설명
-- PASSWORD EXPIRE NEVER: 만료 없음
-- PASSWORD EXPIRE INTERVAL n DAY: n일 유효
-- ACCOUNT LOCK / ACCOUNT UNLOCK: 계정 잠금/해제
-- PASSWORD HISTORY n: 최근 n개 비밀번호 재사용 금지
```

**권한 분류**:

```sql
-- Administrative privileges (전역만 가능)
GRANT SUPER ON *.* TO user@'ip';
GRANT BACKUP_ADMIN ON *.* TO bkpuser@'localhost';

-- Database privileges
GRANT EVENT ON *.* TO user@'ip';
GRANT EVENT ON users1.* TO user@'ip';

-- Object privileges
GRANT SELECT, INSERT, UPDATE, DELETE ON *.* TO user@'ip';
GRANT SELECT, INSERT, UPDATE, DELETE ON db1.* TO user@'ip';
GRANT SELECT, INSERT, UPDATE, DELETE ON db1.t1 TO user@'ip';

-- 역할별 권장 권한
-- 관리자
GRANT ALL PRIVILEGES ON *.* TO bosadmin@'localhost' WITH GRANT OPTION;
GRANT ALL PRIVILEGES ON asia.* TO asiaadmin@'localhost' WITH GRANT OPTION;

-- mysqldump 백업 계정
GRANT LOCK TABLES, SELECT, SHOW VIEW, RELOAD, REPLICATION CLIENT, EVENT, TRIGGER ON *.* TO mysqldump_user@'localhost';

-- xtrabackup 백업 계정
GRANT BACKUP_ADMIN, PROCESS, RELOAD, LOCK TABLES, REPLICATION CLIENT ON *.* TO 'bkpuser'@'localhost';
GRANT SELECT ON performance_schema.log_status TO 'bkpuser'@'localhost';

-- 서비스 계정 (DML/DQL만)
GRANT SELECT, INSERT, UPDATE, DELETE, EXECUTE ON db1.* TO user1@'192.168.%';
```

**sys schema 접근 권한 (root 외 일반 유저)**:
```sql
GRANT SELECT ON *.* TO `mon`@`192.168.%`;
GRANT EXECUTE ON sys.* TO `mon`@`192.168.%`;
GRANT INSERT, UPDATE ON sys.sys_config TO `mon`@`192.168.%`;
```

**권한 확인/회수**:
```sql
SHOW GRANTS FOR 'user'@'%';
REVOKE ALL PRIVILEGES, GRANT OPTION ON *.* FROM 'user'@'%';
FLUSH PRIVILEGES;
```

### 특정 유저 접속 차단 (계정 삭제 없이)

```sql
-- 방법 1: mysql_no_login 플러그인 (권장 - 완전 차단)
UPDATE mysql.user SET plugin='mysql_no_login' WHERE User='blocked_user';
FLUSH PRIVILEGES;
-- 복구: plugin을 'mysql_native_password'로 원복

-- 방법 2: Host 값 변경
UPDATE mysql.user SET Host='invalid_host' WHERE User='blocked_user';
FLUSH PRIVILEGES;

-- 방법 3: 모든 권한 회수 (로그인은 가능, 작업 불가)
REVOKE ALL PRIVILEGES, GRANT OPTION ON *.* FROM 'blocked_user';

-- 방법 4: AFTER CONNECT 트리거 (MySQL 8.0+)
CREATE TRIGGER block_login AFTER CONNECT ON * FOR EACH ROW BEGIN
  IF (USER() LIKE 'blocked_user@%') THEN
    SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Your account has been blocked';
  END IF;
END;
```

| 방법 | 완전 차단 | 복원 가능 |
|------|----------|----------|
| mysql_no_login 플러그인 | ✅ | ✅ |
| Host 값 변경 | ✅ | ✅ |
| 권한 회수 | ❌ (로그인은 가능) | ✅ |
| AFTER CONNECT 트리거 | ✅ | ✅ |

### DB 버전 및 용량 조회

```sql
-- 버전 확인
SELECT version();
SHOW GLOBAL VARIABLES LIKE '%version%';

-- 데이터베이스별 용량
SELECT table_schema,
       SUM(data_length+index_length+data_free)/1024/1024 AS total_mb,
       SUM(data_length)/1024/1024 AS data_mb,
       SUM(index_length)/1024/1024 AS index_mb,
       COUNT(*) AS tables
FROM information_schema.tables
GROUP BY table_schema
ORDER BY 2 DESC;

-- 테이블별 용량 (상위 10개)
SELECT table_schema, table_name,
       (data_length+index_length+data_free)/1024/1024 AS total_mb
FROM information_schema.tables
ORDER BY 3 DESC LIMIT 10;

-- 인덱스 사용량
SELECT database_name, table_name, index_name,
       stat_value * @@innodb_page_size AS index_size_bytes
FROM mysql.innodb_index_stats WHERE stat_name='size';
```

### 오래 수행 중인 TX 찾기 및 분석

```sql
-- 1. Transaction History Length 확인 (MVCC undo log 길이)
SELECT count FROM INFORMATION_SCHEMA.INNODB_METRICS
WHERE NAME='trx_rseg_history_len';
-- 또는
SHOW ENGINE INNODB STATUS \G;   -- TRANSACTIONS 섹션의 History list length

-- 2. 장시간 수행중인 DML (현재 executing 상태)
SELECT b.processlist_user, b.processlist_state,
       SEC_TO_TIME(a.TIMER_WAIT/1000000000000) AS timer_wait,
       IF(LENGTH(a.SQL_TEXT)>64, CONCAT(LEFT(a.SQL_TEXT,30),' ... ',RIGHT(a.SQL_TEXT,30)), a.SQL_TEXT) AS query,
       a.ROWS_AFFECTED, a.ROWS_SENT, a.ROWS_EXAMINED
FROM performance_schema.events_statements_current a,
     performance_schema.threads b
WHERE a.current_schema='DB명'
  AND a.thread_id=b.thread_id AND b.processlist_state='executing'
ORDER BY a.timer_wait DESC LIMIT 10;

-- 또는
SELECT * FROM sys.session WHERE trx_state='ACTIVE' AND current_statement IS NOT NULL AND time > 10;

-- 3. 장시간 수행되었던 DML 이력 (최근 10분, 10초 이상)
SELECT ps_t.THREAD_ID, concat(ps_t.PROCESSLIST_USER,'@',ps_t.PROCESSLIST_HOST) AS db_account,
       ps_esh.SQL_TEXT, sys.format_time(ps_esh.TIMER_WAIT) AS duration,
       date_sub(now(), interval (SELECT variable_value FROM performance_schema.global_status
                                  WHERE variable_name='UPTIME') - ps_esh.TIMER_START*10e-13 second) AS start_time
FROM performance_schema.events_statements_history ps_esh
LEFT OUTER JOIN performance_schema.threads ps_t ON ps_esh.THREAD_ID=ps_t.THREAD_ID
WHERE (SELECT variable_value FROM performance_schema.global_status WHERE variable_name='UPTIME') - ps_esh.TIMER_START*10e-13 < 10*60
  AND ps_esh.TIMER_WAIT*10e-13 > 10
ORDER BY TIMER_START, ps_esh.TIMER_WAIT DESC;

-- 4. 커밋 안 하고 대기 중인 TX (600초 이상)
SELECT is_pl.ID, ps_esh.thread_id,
       concat(is_pl.USER,'@',is_pl.HOST) AS db_account,
       ps_etc.STATE,
       trx.trx_started AS 'transaction begin time',
       TIME_TO_SEC(timediff(now(), trx.trx_started)) AS 'transaction duration',
       ps_esh.SQL_TEXT,
       sys.format_time(ps_esh.TIMER_WAIT) AS 'statement duration'
FROM performance_schema.events_transactions_current ps_etc,
     information_schema.innodb_trx trx,
     performance_schema.events_statements_history ps_esh,
     performance_schema.threads ps_th,
     information_schema.processlist is_pl
WHERE ps_etc.STATE='ACTIVE'
  AND ps_etc.THREAD_ID=ps_th.THREAD_ID
  AND ps_etc.THREAD_ID=ps_esh.THREAD_ID
  AND ps_esh.EVENT_ID >= ps_etc.NESTING_EVENT_ID
  AND ps_th.PROCESSLIST_ID=is_pl.ID
  AND trx.trx_mysql_thread_id=ps_th.PROCESSLIST_ID
  AND TIME_TO_SEC(timediff(now(), trx.trx_started)) > 600
ORDER BY ps_esh.thread_id, ps_esh.event_id;

-- 5. 대량 변경 작업 중인 쿼리
SELECT 'session table' AS source,
       conn_id AS processlist_id, user AS userhost, db, state,
       statement_latency AS time_wait,
       ifnull(current_statement, last_statement) AS query,
       rows_affected, rows_examined, rows_sent
FROM sys.session WHERE trx_state='ACTIVE' AND lower(current_statement) NOT LIKE 'select%';
```

### 기 실행된 쿼리 내역 확인

```sql
-- 특정 세션(processlist_id)의 현재 수행 쿼리
SELECT esh.THREAD_ID, t.processlist_id,
       SEC_TO_TIME(esh.timer_wait/1000000000000) AS exec_time,
       sys.format_time(esh.timer_wait) AS 'work time',
       esh.SQL_TEXT
FROM performance_schema.events_statements_current esh
JOIN performance_schema.threads t ON t.thread_id=esh.thread_id
WHERE t.processlist_id=56604;

-- 특정 세션의 수행 이력 (events_statements_history_long)
SELECT esh.THREAD_ID, t.processlist_id,
       ROUND(esh.timer_wait*10E-10, 3) AS `wait in(ms)`,
       sys.format_time(esh.timer_wait) AS 'wait time',
       esh.SQL_TEXT
FROM performance_schema.events_statements_history_long esh
JOIN performance_schema.threads t ON t.thread_id=esh.thread_id
WHERE t.processlist_id=56604
  AND esh.CURRENT_SCHEMA='DB명'
ORDER BY TIMER_END ASC LIMIT 10 \G;
```

### 쿼리 로깅

**General Log (모든 쿼리 - 진단용, 부하 큼)**:
```sql
-- 현재 설정 확인
SHOW VARIABLES LIKE '%general%';

-- 켜기/끄기
SET GLOBAL general_log='ON';
SET GLOBAL general_log='OFF';

-- 출력 대상 변경
SET GLOBAL LOG_OUTPUT='TABLE';      -- DB 테이블로 저장
SET GLOBAL LOG_OUTPUT='FILE';       -- 파일로 저장
SET GLOBAL LOG_OUTPUT='FILE,TABLE'; -- 양쪽에 동시 저장

-- 테이블에 기록된 쿼리 조회
SELECT *, CONVERT(argument USING utf8) FROM mysql.general_log LIMIT 10;
```

**Slow Query Log**:
```sql
-- 현재 설정 확인
SHOW VARIABLES LIKE 'slow_query_%';

-- 켜기/끄기
SET GLOBAL slow_query_log='ON';
SET GLOBAL slow_query_log='OFF';

-- 기준 시간 설정
SET GLOBAL long_query_time=3;      -- 3초 이상
SET GLOBAL long_query_time=0;      -- 모든 쿼리 로깅

-- 테이블에 기록된 slow 쿼리 조회
SELECT *, CONVERT(sql_text USING utf8) FROM mysql.slow_log LIMIT 10;

-- slow_log 테이블 truncate
SET GLOBAL slow_query_log='OFF';
TRUNCATE TABLE mysql.slow_log;
SET GLOBAL slow_query_log='ON';
```

**Percona Slow Query 확장 파라미터**:
```ini
# my.cnf
slow_query_log           = ON
long_query_time          = 3
log_slow_rate_limit      = 1        # 1: 모든 쿼리 로깅
log_slow_rate_type       = query
log_slow_verbosity       = full     # 실행계획 포함
log_slow_admin_statements= ON
slow_query_log_always_write_time = 1  # 1초 초과 시 무조건 기록
slow_query_log_use_global_control= all
```

### Audit Log (Percona Audit Plugin)

```sql
-- 플러그인 설치
INSTALL PLUGIN audit_log SONAME 'audit_log.so';
SHOW VARIABLES LIKE 'audit%';

-- Policy 설정
-- ALL: 모든 event
-- LOGINS: 로그인 관련만
-- QUERIES: 쿼리만
-- NONE: 비활성화
SET GLOBAL audit_log_policy='QUERIES';

-- 특정 계정만 감사 (include/exclude 중 하나만 설정 가능)
SET GLOBAL audit_log_include_accounts='user1@localhost,root@%';
-- audit_log_exclude_accounts와 동시 설정 불가 → 오류 발생

-- File rotation 설정
-- audit_log_rotate_on_size: 파일 최대 크기
-- audit_log_rotations: 보관 파일 수
```

**my.cnf Audit 설정**:
```ini
[mysqld]
audit_log_rotations          = 5
audit_log_rotate_on_size     = 1073741824  # 1GB
audit_log_include_accounts   = 'root@%,root@localhost'
audit_log_file               = /home/data/trc/audit.log
audit_log_policy             = QUERIES
```
