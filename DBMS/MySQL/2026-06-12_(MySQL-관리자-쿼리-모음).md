## 1. 핵심 요약
- MySQL 8.0 User 생성 시 인증방식(`mysql_native_password` vs `caching_sha2_password`), 비밀번호 정책, 계정 잠금 옵션을 명시하는 것이 권장됨.
- 오래된 TX 추적: `information_schema.innodb_trx` + `trx_started` 기준.
- `general_log`는 모든 쿼리를 기록하지만 부하가 크므로 일시적 진단에만 사용.

## 2. 상세 설명

### User 생성 및 권한 관리

```sql
-- 전체 옵션을 명시한 계정 생성
CREATE USER 'user'@'%'
   IDENTIFIED WITH 'mysql_native_password' BY 'password'
   REQUIRE NONE
   PASSWORD EXPIRE INTERVAL 30 DAY
   ACCOUNT UNLOCK
   PASSWORD HISTORY DEFAULT
   PASSWORD REUSE INTERVAL DEFAULT
   PASSWORD REQUIRE CURRENT DEFAULT;

-- 비밀번호 정책
-- PASSWORD EXPIRE NEVER: 만료 없음
-- PASSWORD EXPIRE INTERVAL n DAY: n일 유효
-- ACCOUNT LOCK / ACCOUNT UNLOCK: 계정 잠금/해제
```

**권한 부여·조회**:
```sql
-- DB 전체 권한
GRANT ALL PRIVILEGES ON DB명.* TO 'user'@'%';

-- 특정 권한
GRANT SELECT, INSERT, UPDATE ON DB명.* TO 'user'@'%';

-- 권한 확인
SHOW GRANTS FOR 'user'@'%';

-- 권한 회수
REVOKE ALL PRIVILEGES ON DB명.* FROM 'user'@'%';

FLUSH PRIVILEGES;
```

**sys schema 접근 권한**:
```sql
GRANT EXECUTE ON sys.* TO 'user'@'%';
GRANT SELECT ON performance_schema.* TO 'user'@'%';
```

### 세션·트랜잭션 모니터링

**오래 수행 중인 TX 찾기**:
```sql
SELECT trx_id, trx_state, trx_started,
       TIME_TO_SEC(TIMEDIFF(NOW(), trx_started)) AS elapsed_sec,
       trx_mysql_thread_id, trx_query
FROM information_schema.innodb_trx
WHERE trx_state = 'RUNNING'
ORDER BY trx_started;
```

**COMMIT 안 하고 대기 중인 세션**:
```sql
SELECT * FROM sys.session WHERE trx_state = 'ACTIVE';
-- 또는
SELECT * FROM information_schema.innodb_trx
WHERE TIME_TO_SEC(TIMEDIFF(NOW(), trx_started)) > 60 \G;
```

**현재 실행 중인 쿼리**:
```sql
SHOW FULL PROCESSLIST;
-- 또는
SELECT * FROM information_schema.PROCESSLIST WHERE COMMAND != 'Sleep';
```

**기 실행된 쿼리 내역 (events_statements_history)**:
```sql
SELECT ps_t.processlist_id,
       concat(ps_t.PROCESSLIST_USER,'@',ps_t.PROCESSLIST_HOST) AS db_account,
       ps_esh.SQL_TEXT,
       sys.format_time(ps_esh.TIMER_WAIT) AS duration
FROM performance_schema.events_statements_history ps_esh
INNER JOIN performance_schema.threads ps_t ON ps_t.thread_id = ps_esh.thread_id
WHERE ps_t.processlist_id = <PID>
  AND ps_esh.SQL_TEXT IS NOT NULL
  AND ps_esh.MYSQL_ERRNO = 0
ORDER BY ps_esh.TIMER_START \G;
```

**에러를 일으키는 클라이언트 프로세스 찾기**:
```sql
SELECT * FROM performance_schema.events_errors_summary_global_by_error
ORDER BY SUM_ERROR_RAISED DESC LIMIT 30;
```

### 접속 차단

```sql
-- 계정 잠금
ALTER USER 'user'@'%' ACCOUNT LOCK;

FLUSH HOSTS;

KILL <processlist_id>;
```

### 쿼리 로깅

**General Log (모든 쿼리 기록 - 일시적 진단용)**:
```sql
-- 활성화 (부하 주의)
SET GLOBAL general_log = ON;
SET GLOBAL general_log_file = '/home/data/trc/mysql-general.log';

-- 비활성화
SET GLOBAL general_log = OFF;
```

**Slow Query Log**:
```ini
# my.cnf
slow_query_log           = ON
long_query_time          = 3         # 3초 이상
log_slow_verbosity       = full
log_slow_admin_statements= ON
slow_query_log_file      = /home/data/trc/mysql-query.log
```

### DB 상태 종합 확인 쿼리

```sql
-- 전체 DB 크기
SELECT datname, pg_size_pretty(pg_database_size(datname)) FROM pg_database;  -- PG용
SELECT table_schema, ROUND(SUM(data_length+index_length)/1024/1024,1) AS MB
FROM information_schema.tables GROUP BY table_schema;  -- MySQL용

-- 객체 유형별 개수
SELECT * FROM sys.schema_object_overview WHERE db='DB명';

-- 변경 없는 테이블 확인
SELECT t.table_schema, t.table_name
FROM information_schema.tables t
JOIN performance_schema.table_io_waits_summary_by_table tio
  ON tio.object_schema=t.table_schema AND tio.object_name=t.table_name
WHERE t.table_schema NOT IN ('mysql','performance_schema','sys')
  AND tio.count_write = 0
ORDER BY t.table_schema, t.table_name;
```
