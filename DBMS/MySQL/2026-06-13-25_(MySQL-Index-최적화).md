## 1. 핵심 요약

적절한 인덱스는 쿼리 성능을 10배 이상 향상시킵니다.
선택도(Cardinality), 쿼리 조건, 정렬 순서를 고려하여 복합 인덱스를 설계합니다.

---

## 2. 인덱스 설계

```sql
-- 단일 컬럼
CREATE INDEX idx_user_id ON orders(user_id);

-- 복합 인덱스 (WHERE, JOIN 순서 중요)
CREATE INDEX idx_user_date ON orders(user_id, order_date);

-- FULLTEXT 인덱스
CREATE FULLTEXT INDEX ft_title ON products(title);
```

---

## 3. 미사용 인덱스 정리

```sql
SELECT * FROM sys.schema_unused_indexes
WHERE object_schema != 'mysql';
```

---

## 4. EXPLAIN 분석

```sql
EXPLAIN SELECT * FROM orders WHERE user_id = 1 AND order_date > '2026-01-01';

-- 확인: type=ref 또는 range (ALL이 아니어야 함)
-- rows < table total rows
-- key가 인덱스명
```
