# MySQL 파티션 관리 (RANGE Partition)

- **카테고리**: #DBMS #MySQL #Schema
- **태그**: #partition #RANGE #exchange #reorganize
- **작성일**: 2026-06-13
- **참조 원본**: [[2026-06-12-mysql partition 관리 - BigDataTeam]]

## 1. 핵심 요약

RANGE 파티션은 `PARTITION BY RANGE(컬럼)`으로 구간 분할하며, **MAXVALUE 파티션을 `REORGANIZE`로 쪼개 신규 구간을 추가**합니다(파티션은 추가 개념만 존재).
`EXCHANGE PARTITION`으로 파티션↔테이블을 교체해 대량 데이터를 빠르게 이동/아카이빙할 수 있습니다.

---

## 2. 파티션 테이블 생성

```sql
CREATE TABLE `T1` (
  `symbol` varchar(45) NOT NULL,
  `unix_time` int NOT NULL,
  PRIMARY KEY(`symbol`,`unix_time`)
) ENGINE=InnoDB
PARTITION BY RANGE (unix_time) (
  PARTITION p202001 VALUES LESS THAN (1577836800),
  PARTITION p202002 VALUES LESS THAN (1580515200),
  PARTITION p999999 VALUES LESS THAN MAXVALUE
);
```

> 기존 테이블에 추가: `ALTER TABLE T1 PARTITION BY RANGE(unix_time) ( ... );`
> ⚠️ 파티션 키는 **PK/유니크 키에 포함**되어야 함.

---

## 3. 파티션 조회

```sql
SELECT TABLE_NAME, PARTITION_NAME, PARTITION_METHOD,
       PARTITION_EXPRESSION, PARTITION_DESCRIPTION
FROM information_schema.PARTITIONS WHERE TABLE_NAME = 'T1';

-- 특정 파티션 데이터만 조회
SELECT * FROM T1 PARTITION (p202001);
```

---

## 4. 파티션 추가 (REORGANIZE)

```sql
-- 파티션은 '추가' 개념만 → MAXVALUE 파티션을 쪼개서 신규 구간 추가
ALTER TABLE T1 REORGANIZE PARTITION p999999 INTO (
  PARTITION p202103 VALUES LESS THAN (1583020800),
  PARTITION p999999 VALUES LESS THAN MAXVALUE
);
```

---

## 5. 파티션 삭제

```sql
-- 파티션 + 데이터 삭제
ALTER TABLE T1 DROP PARTITION p202103;

-- 파티셔닝만 제거 (데이터는 유지)
ALTER TABLE T1 REMOVE PARTITIONING;
```

---

## 6. 파티션 EXCHANGE (테이블 교체)

```sql
-- 동일 구조 테이블 생성 후 파티션과 교체
ALTER TABLE T1 EXCHANGE PARTITION p202002 WITH TABLE T1_P202002;
-- → T1의 p202002 데이터가 T1_P202002로 이동(빈 파티션 ↔ 채운 테이블)
ALTER TABLE T1 DROP PARTITION p202002;   -- 비워진 파티션 정리
```

> 대량 데이터 아카이빙/이관 시 INSERT...SELECT 없이 메타데이터 교체로 즉시 처리.
