## 1. 핵심 요약

MySQL 관리자가 일상적으로 사용하는 기본 쿼리 모음입니다.
버전 확인, DB/테이블 용량 조회, 인덱스 분석, 파티션 관리 등을 포함합니다.

## 2. 버전 및 환경 확인

### 2.1 MySQL 버전 조회

```bash
# CLI에서 확인
mysql --version
# 출력: mysql  Ver 14.14 Distrib 5.7.27, for linux-glibc2.12 (x86_64)
```

### 2.2 SQL로 버전 확인

```sql
-- 간단한 버전 확인
SELECT VERSION();
-- 출력: 5.7.27-log

-- 상세 버전 정보
SHOW GLOBAL VARIABLES LIKE '%version%';

-- 주요 항목:
-- version: 5.7.27-log                       (버전)
-- version_comment: MySQL Community Server   (배포판)
-- version_compile_os: linux-glibc2.12      (컴파일 OS)
-- version_compile_machine: x86_64          (아키텍처)
-- innodb_version: 5.7.27                   (InnoDB 버전)
```

## 3. 용량 조회

### 3.1 데이터베이스별 용량

```sql
SELECT
    table_schema,
    SUM(data_length + index_length + data_free)/1024/1024 AS total_mb,
    SUM(data_length)/1024/1024 AS data_mb,
    SUM(index_length)/1024/1024 AS index_mb,
    SUM(data_free)/1024/1024 AS free_mb,
    COUNT(*) AS tables,
    CURDATE() AS today 
FROM
    information_schema.tables
GROUP BY table_schema
ORDER BY 2 DESC;
```

### 3.2 테이블별 용량

```sql
SELECT
    table_schema,
    table_name,
    (data_length + index_length + data_free)/1024/1024 AS total_mb,
    (data_length)/1024/1024 AS data_mb,
    (index_length)/1024/1024 AS index_mb,
    (data_free)/1024/1024 AS free_mb,
    TABLE_ROWS,
    CURDATE() AS today
FROM
    information_schema.tables
WHERE table_schema NOT IN ('mysql', 'information_schema', 'performance_schema', 'sys')
ORDER BY 3 DESC
LIMIT 50;
```

### 3.3 파티션 테이블 용량

```sql
SELECT
    TABLE_SCHEMA,
    TABLE_NAME,
    PARTITION_NAME,
    PARTITION_DESCRIPTION,
    TABLE_ROWS,
    ROUND(DATA_LENGTH/(1024*1024), 2) AS 'DATA_SIZE(MB)',
    ROUND(INDEX_LENGTH/(1024*1024), 2) AS 'INDEX_SIZE(MB)'
FROM
    INFORMATION_SCHEMA.PARTITIONS
WHERE PARTITION_NAME IS NOT NULL
AND TABLE_SCHEMA='target_db'
ORDER BY DATA_LENGTH DESC;
```

## 4. 인덱스 분석

### 4.1 인덱스 사용 횟수

```sql
SELECT 
    object_schema,
    object_name,
    index_name,
    count_star AS usage_count,
    count_read,
    count_write,
    count_delete
FROM 
    performance_schema.table_io_waits_summary_by_index_usage
WHERE object_schema='target_db' AND object_name='target_table'
ORDER BY count_star DESC;
```

### 4.2 사용되지 않는 인덱스

```sql
-- sys 스키마 활용 (MySQL 5.7+)
SELECT *
FROM sys.schema_unused_indexes
WHERE object_schema='target_db'
ORDER BY object_name;

-- 또는 성능 스키마로 직접 쿼리
SELECT 
    object_schema,
    object_name,
    index_name
FROM performance_schema.table_io_waits_summary_by_index_usage
WHERE count_star=0
AND object_schema NOT IN ('mysql', 'information_schema', 'performance_schema')
ORDER BY object_schema, object_name;
```

### 4.3 인덱스 크기

```sql
SELECT 
    database_name,
    table_name,
    index_name,
    ROUND(stat_value*@@innodb_page_size/1024/1024, 2) AS index_size_mb
FROM mysql.innodb_index_stats
WHERE stat_name='size'
AND database_name='target_db'
ORDER BY stat_value DESC;
```

## 5. 객체 검색

### 5.1 Primary Key 없는 테이블

```sql
SELECT 
    tab.table_schema,
    tab.table_name 
FROM 
    information_schema.tables tab
LEFT JOIN 
    information_schema.table_constraints tco 
    ON tab.table_schema = tco.table_schema
    AND tab.table_name = tco.table_name 
    AND tco.constraint_type = 'PRIMARY KEY'
WHERE 
    tco.constraint_type IS NULL
    AND tab.table_schema NOT IN ('mysql', 'information_schema', 'performance_schema', 'sys')
    AND tab.table_type = 'BASE TABLE'
ORDER BY 
    tab.table_schema, tab.table_name;
```

### 5.2 파티션 테이블 목록

```sql
SELECT DISTINCT
    TABLE_SCHEMA,
    TABLE_NAME,
    PARTITION_METHOD,
    SUBPARTITION_METHOD
FROM 
    INFORMATION_SCHEMA.PARTITIONS
WHERE PARTITION_NAME IS NOT NULL
ORDER BY TABLE_SCHEMA, TABLE_NAME;
```

### 5.3 특정 컬럼 검색

```sql
SELECT 
    TABLE_SCHEMA,
    TABLE_NAME,
    COLUMN_NAME,
    COLUMN_TYPE,
    IS_NULLABLE
FROM 
    INFORMATION_SCHEMA.COLUMNS
WHERE 
    TABLE_SCHEMA='target_db'
    AND COLUMN_NAME LIKE '%search_term%'
ORDER BY TABLE_NAME, ORDINAL_POSITION;
```

## 6. 성능 분석

### 6.1 slow query log 확인

```sql
-- Slow Query Log가 활성화되었는지 확인
SHOW VARIABLES LIKE 'slow_query_log%';

-- Slow Query Log 테이블 조회
SELECT 
    query_time,
    lock_time,
    rows_sent,
    rows_examined,
    sql_text
FROM mysql.slow_log
WHERE start_time > DATE_SUB(NOW(), INTERVAL 1 HOUR)
ORDER BY query_time DESC
LIMIT 20;
```

### 6.2 General Log 크기

```sql
-- General Log 테이블 조회
SELECT 
    COUNT(*) AS total_rows,
    ROUND(DATA_LENGTH/1024/1024, 2) AS size_mb
FROM INFORMATION_SCHEMA.TABLES
WHERE TABLE_SCHEMA='mysql' AND TABLE_NAME='general_log';
```

## 7. 메모리 관련 설정

### 7.1 버퍼풀 크기

```sql
SHOW VARIABLES LIKE 'innodb_buffer_pool_size';
```

### 7.2 메모리 사용량 조회

```sql
SELECT 
    performance_schema.threads.THREAD_ID,
    COUNT_ALLOC,
    COUNT_FREE,
    MEMORY_ALLOC,
    MEMORY_FREE
FROM performance_schema.memory_summary_by_thread_by_event_name
WHERE USER_NAME IS NOT NULL
ORDER BY MEMORY_ALLOC DESC
LIMIT 10;
```

## 8. 연결 관리

### 8.1 현재 연결 조회

```sql
SHOW PROCESSLIST;
-- 또는
SELECT * FROM INFORMATION_SCHEMA.PROCESSLIST;
```

### 8.2 특정 사용자 연결만 조회

```sql
SELECT 
    ID,
    USER,
    HOST,
    DB,
    TIME,
    STATE,
    INFO
FROM INFORMATION_SCHEMA.PROCESSLIST
WHERE USER='app_user'
ORDER BY TIME DESC;
```

### 8.3 연결 종료

```sql
-- 특정 연결 ID 종료
KILL 123;

-- 사용자의 모든 연결 종료
KILL SESSION FOR USER 'app_user';
```

## 9. 권한 관리

### 9.1 사용자 목록

```sql
SELECT 
    User,
    Host,
    Super_priv,
    Grant_priv,
    Account_locked
FROM mysql.user
ORDER BY User, Host;
```

### 9.2 특정 사용자 권한

```sql
SHOW GRANTS FOR 'user'@'localhost';
```

### 9.3 권한 부여

```sql
GRANT SELECT, INSERT, UPDATE ON database.* TO 'user'@'%';
FLUSH PRIVILEGES;
```
