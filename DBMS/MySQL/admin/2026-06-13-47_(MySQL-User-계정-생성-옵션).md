## 1. 핵심 요약

`CREATE USER`는 인증 방식·SSL 요구·비밀번호 만료/이력/재사용/변경정책·계정 잠금을 한 번에 지정할 수 있습니다.
운영에서는 `IDENTIFIED WITH`(인증 플러그인), `PASSWORD EXPIRE INTERVAL`(만료), `PASSWORD HISTORY`/`REUSE INTERVAL`(재사용 방지)을 조합해 비밀번호 정책을 강제합니다.

---

## 2. 생성 예시

```sql
CREATE USER 'user'@'%'
   IDENTIFIED WITH 'mysql_native_password' BY 'password'
   REQUIRE NONE
   PASSWORD EXPIRE INTERVAL 30 DAY
   ACCOUNT UNLOCK
   PASSWORD HISTORY DEFAULT
   PASSWORD REUSE INTERVAL DEFAULT
   PASSWORD REQUIRE CURRENT DEFAULT;
```

---

## 3. 옵션 상세

| 옵션 | 설명 |
|------|------|
| **IDENTIFIED WITH** | 인증 방식(플러그인)과 비밀번호 설정 |
| **REQUIRE** | 접속 시 SSL/TLS 암호화 채널 요구 여부 (`NONE`/`SSL`/`X509` 등) |
| **PASSWORD EXPIRE** | 비밀번호 유효기간 |
| **ACCOUNT LOCK/UNLOCK** | 계정 잠금/해제 |

### 3.1 PASSWORD EXPIRE
- `PASSWORD EXPIRE`: 생성과 동시에 즉시 만료
- `NEVER`: 만료 없음
- `DEFAULT`: `default_password_lifetime` 시스템 변수 따름
- `INTERVAL n DAY`: 오늘부터 n일 후 만료

### 3.2 PASSWORD HISTORY (재사용 이력)
- `DEFAULT`: `password_history` 변수 개수만큼 이력 저장
- `n`: 최근 n개 비밀번호 재사용 금지

### 3.3 PASSWORD REUSE INTERVAL (재사용 금지 기간)
- `DEFAULT`: `password_reuse_interval` 변수 따름
- `n DAY`: n일 이후 재사용 허용

### 3.4 PASSWORD REQUIRE (변경 시 현재 비밀번호 요구)
- `CURRENT`: 변경 시 현재(만료된) 비밀번호 입력 필요
- `OPTIONAL`: 입력 불필요
- `DEFAULT`: `password_require_current` 변수 따름

### 3.5 ACCOUNT LOCK / UNLOCK
- `LOCK`: 계정 사용 불가(잠금) — `ALTER USER`로도 변경 가능
- `UNLOCK`: 잠금 해제
