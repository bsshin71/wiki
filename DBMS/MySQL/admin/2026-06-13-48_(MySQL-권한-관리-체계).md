## 1. 핵심 요약

MySQL 권한은 **범위(전역/DB/객체)**, **특성(Static/Dynamic)**, **역할(관리자/백업관리자/서비스사용자)** 세 축으로 분류됩니다.
전역 권한은 `ON *.*`로만 부여 가능하며, Dynamic 권한은 서버 기동 시점/플러그인 설치에 따라 사용 가능 여부가 결정됩니다.

---

## 2. 범위에 따른 분류

| 분류 | 적용 범위 | 예시 |
|------|-----------|------|
| **Administrative** | 서버 전역(특정 DB 무관) | `GRANT SUPER ON *.* TO user@'ip';` |
| **Database** | 특정 DB 및 내부 객체 | `GRANT EVENT ON db1.* TO user@'ip';` |
| **Object** | 모든/특정 DB, 특정 객체 | `GRANT SELECT,INSERT,UPDATE,DELETE ON db1.t1 TO user@'ip';` |

> 전역 권한은 `ON '*.*'` 만 사용 가능.

---

## 3. 특성에 따른 분류

### 3.1 Static privileges (서버 builtin)
서버에 내장되어 기동 중이면 제한 없이 사용. 주요 항목:

| Privilege | Context |
|-----------|---------|
| ALL [PRIVILEGES] | 모든 권한 동의어 |
| SELECT / INSERT / UPDATE / DELETE | Tables/Columns |
| CREATE / DROP / ALTER / INDEX | Databases·Tables |
| CREATE USER / CREATE ROLE / DROP ROLE | Server administration |
| GRANT OPTION | 권한 위임 |
| PROCESS / RELOAD / SHUTDOWN / SUPER | Server administration |
| REPLICATION CLIENT / REPLICATION SLAVE | Server administration |
| LOCK TABLES / FILE / TRIGGER / EVENT | 각 Context |
| USAGE | "권한 없음" 동의어 |

### 3.2 Dynamic privileges (기동 시/플러그인 의존)
조건·플러그인 설치에 따라 사용 가능. 주요 항목:

| Privilege | Context |
|-----------|---------|
| BACKUP_ADMIN | 백업 관리 |
| BINLOG_ADMIN / BINLOG_ENCRYPTION_ADMIN | 백업·복제 관리 |
| CLONE_ADMIN | Clone 관리 |
| CONNECTION_ADMIN | 서버 관리(연결) |
| GROUP_REPLICATION_ADMIN / REPLICATION_SLAVE_ADMIN | 복제 관리 |
| INNODB_REDO_LOG_ARCHIVE | redo log 아카이빙 |
| AUDIT_ADMIN | Audit log 관리 |
| SYSTEM_VARIABLES_ADMIN / SESSION_VARIABLES_ADMIN | 변수 관리 |
| SYSTEM_USER / SET_USER_ID / ROLE_ADMIN | 서버 관리 |
| FLUSH_TABLES / FLUSH_STATUS / FLUSH_OPTIMIZER_COSTS | 서버 관리 |

## 4. 역할에 따른 분류

### 4.1 관리자
```sql
-- 전역 관리자(별도 생성 권장, root 노출 회피)
GRANT ALL PRIVILEGES ON *.* TO bosadmin@'localhost' WITH GRANT OPTION;
-- 특정 DB 관리자
GRANT ALL PRIVILEGES ON asia.* TO asiaadmin@'localhost' WITH GRANT OPTION;
FLUSH PRIVILEGES;
```
> 관리자 계정의 접속 허용 IP는 관례적으로 `localhost`, `127.0.0.1`만 허용하기도 함.

### 4.2 백업 관리자 (도구별 최소 권한)
```sql
-- mysqldump
GRANT LOCK TABLES, SELECT, SHOW VIEW, RELOAD, REPLICATION CLIENT, EVENT, TRIGGER
  ON *.* TO mysqldump_user@'localhost';

-- xtrabackup
GRANT BACKUP_ADMIN, PROCESS, RELOAD, LOCK TABLES, REPLICATION CLIENT
  ON *.* TO 'bkpuser'@'localhost';
GRANT SELECT ON performance_schema.log_status TO 'bkpuser'@'localhost';
```

### 4.3 서비스 사용자 (DML/DQL + procedure)
```sql
GRANT SELECT, INSERT, UPDATE, DELETE, EXECUTE ON db1.* TO user1@'192.168.%';
```
