## 1. 핵심 요약

계정을 삭제하거나 비밀번호를 바꾸지 않고 접속만 막는 5가지 방법이 있습니다.
가장 간단한 방법은 **`mysql_no_login` 플러그인 전환**이고, 보다 강력한 차단은 **`AFTER CONNECT` 트리거**(8.0+)입니다. 모두 복원이 가능합니다.

---

## 2. 방법별 정리

### 2.1 mysql_no_login 플러그인 (권장·간단)
```sql
UPDATE mysql.user SET plugin = 'mysql_no_login' WHERE User = 'blocked_user';
FLUSH PRIVILEGES;
```
- 로그인 자체 차단. 복원은 `plugin`을 원래 값(`mysql_native_password` 등)으로 되돌림.

### 2.2 Host 값 변경
```sql
UPDATE mysql.user SET Host = 'invalid_host' WHERE User = 'blocked_user';
FLUSH PRIVILEGES;
```
- 존재하지 않는 호스트라 매칭 실패 → 로그인 불가. 기존 Host 복원 시 해제.

### 2.3 모든 권한 회수 (로그인은 가능)
```sql
REVOKE ALL PRIVILEGES, GRANT OPTION ON *.* FROM 'blocked_user';
FLUSH PRIVILEGES;
```
- 로그인은 되지만 아무 작업 불가. **접속 자체는 막지 못함**.

### 2.4 AFTER CONNECT 트리거 (8.0+, 강력)
```sql
CREATE DATABASE IF NOT EXISTS security;
USE security;
CREATE TABLE IF NOT EXISTS block_users (user VARCHAR(50) PRIMARY KEY);
INSERT INTO block_users VALUES ('blocked_user');

DELIMITER //
CREATE TRIGGER block_login AFTER CONNECT ON * FOR EACH ROW
BEGIN
    IF (USER() LIKE 'blocked_user@%') THEN
        SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Your account has been blocked';
    END IF;
END;
//
DELIMITER ;
```
- 로그인 시도 시 `45000` 오류 강제 발생.

### 2.5 my.cnf 차단 (전역/네트워크 단위)
```ini
[mysqld]
skip-networking          # 모든 원격 접속 차단
# 또는
bind-address = 127.0.0.1 # 로컬만 허용, 원격 차단
```

## 3. 비교 및 추천

| 방법 | 계정 삭제 없이 차단 | 복원 가능 |
|------|:---:|:---:|
| mysql_no_login 플러그인 | ✅ | ✅ |
| Host 값 변경 | ✅ | ✅ |
| 모든 권한 제거 | 🚫(로그인 가능) | ✅ |
| AFTER CONNECT 트리거 | ✅ | ✅ |
| my.cnf 차단 | ✅(전역) | ✅ |

- 가장 쉬운 방법: **mysql_no_login 플러그인**
- 강력한 차단: **AFTER CONNECT 트리거**
