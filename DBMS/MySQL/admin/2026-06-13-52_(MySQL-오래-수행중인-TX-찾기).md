## 1. 핵심 요약

InnoDB의 **History List Length(HLL)**는 undo log에 보관된 옛 버전(MVCC snapshot)의 총합으로, 길어지면 쿼리·종료 속도 저하와 purge 시 CPU 급등을 유발합니다.
장기 실행/미커밋 트랜잭션이 주원인이며, Performance Schema + `INNODB_TRX`로 장시간 수행·미커밋 트랜잭션을 추적합니다.

---

## 2. History Length란

- MVCC는 롤백 대비로 각 행의 여러 버전을 **undo log**에 보관.
- **History List Length** = undo log가 관리 중인 옛 버전(변경 되돌림 데이터)의 총합.

### 늘어나는 이유
- 장기 실행 트랜잭션(읽기/쓰기), heavy write load.

### 길어질 때 문제
- 이전 행 버전 과다 → 쿼리·DB 종료 속도 저하
- purge 시 순간적 CPU 급등
- 장기 유지 시 disk flush로 write 부하 증가

## 3. History Length 체크

```sql
-- 1) SHOW ENGINE INNODB STATUS의 TRANSACTIONS 섹션
--    History list length 0

-- 2) INNODB_METRICS
SELECT COUNT FROM INFORMATION_SCHEMA.INNODB_METRICS
 WHERE NAME = 'trx_rseg_history_len';
```

## 4. 장시간 수행중인 DML 조회

```sql
SELECT b.processlist_user, b.processlist_state,
       SEC_TO_TIME(a.TIMER_WAIT/1000000000000) AS timer_wait,
       a.SQL_TEXT, a.ROWS_AFFECTED, a.ROWS_SENT, a.ROWS_EXAMINED
FROM performance_schema.events_statements_current a,
     performance_schema.threads b
WHERE a.thread_id = b.thread_id AND b.processlist_state = 'executing'
ORDER BY a.timer_wait DESC LIMIT 10;

-- 또는 sys.session
SELECT * FROM sys.session
 WHERE trx_state='ACTIVE' AND current_statement IS NOT NULL AND time > 10;
```

## 5. 장시간 수행되었던 DML 조회 (최근 10분 내 10초 이상)

```sql
SELECT ps_t.THREAD_ID,
       CONCAT(ps_t.PROCESSLIST_USER,'@',ps_t.PROCESSLIST_HOST) AS db_account,
       ps_esh.SQL_TEXT, sys.format_time(ps_esh.TIMER_WAIT) AS duration
FROM performance_schema.events_statements_history ps_esh
  LEFT JOIN performance_schema.threads ps_t ON ps_esh.THREAD_ID = ps_t.THREAD_ID
WHERE (SELECT variable_value FROM performance_schema.global_status WHERE variable_name='UPTIME')
        - ps_esh.TIMER_START*10e-13 < 10*60
  AND ps_esh.TIMER_WAIT*10e-13 > 10
ORDER BY TIMER_START, ps_esh.TIMER_WAIT DESC;
```

## 6. 장시간 미커밋 트랜잭션 조회 (10분 이상)

```sql
-- events_transactions_current(STATE=ACTIVE) + innodb_trx + processlist 조인
-- TIME_TO_SEC(timediff(now(), trx.trx_started)) > 600
```

간단 버전:

```sql
-- 60초 이상 수행 중인 트랜잭션
SELECT * FROM information_schema.innodb_trx
 WHERE TIME_TO_SEC(timediff(now(), trx_started)) > 60\G

-- trx_mysql_thread_id로 프로세스 확인
SELECT * FROM information_schema.processlist WHERE id = 310318;
```

## 7. 대량 변경작업 쿼리 조회

```sql
-- 현재 ACTIVE & non-select
SELECT conn_id AS processlist_id, user, db, state,
       statement_latency AS time_wait,
       IFNULL(current_statement, last_statement) AS query,
       rows_affected, rows_examined, rows_sent
FROM sys.session
WHERE trx_state='ACTIVE' AND LOWER(current_statement) NOT LIKE 'select%'
ORDER BY rows_affected DESC;
```
