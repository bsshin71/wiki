## 1. 핵심 요약

Performance Schema의 **에러 요약 테이블 → 소켓 인스턴스 테이블**을 따라가면 오류를 많이 일으키는 세션의 thread_id, 그리고 그 클라이언트의 **IP·port**까지 역추적할 수 있습니다.
마지막으로 해당 서버에서 `netstat`으로 그 port를 쓰는 OS 프로세스를 특정합니다.

---

## 2. 추적 3단계

### 2.1 에러 발생 thread_id 확인
```sql
SELECT * FROM performance_schema.events_errors_summary_by_thread_by_error
 ORDER BY sum_error_raised DESC LIMIT 100;
```

### 2.2 thread_id → 클라이언트 IP·port 확인
```sql
SELECT * FROM performance_schema.socket_instances
 WHERE thread_id = 257177;
```

### 2.3 클라이언트 서버에서 OS 프로세스 특정
```bash
# 해당 IP 서버에 로그인 후, 위에서 확인한 port 사용 프로세스 조회
netstat -atlup | grep <port>
```
