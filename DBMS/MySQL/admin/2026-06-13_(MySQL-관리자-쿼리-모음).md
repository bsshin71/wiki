## 1. 핵심 요약

MySQL 일상 운영에 필요한 **사용자·권한 관리, 세션 모니터링, 로깅, 감사** 쿼리 모음입니다.
이 문서의 쿼리들은 Performance Schema, information_schema, sys 스키마를 활용한 현황 파악 및 문제 해결용입니다.
정기적으로 실행하면 데이터베이스 건강도를 유지할 수 있습니다.

## 2. 사용자 관리

### 2.1 사용자 생성

```sql
-- 기본 형식 (MySQL 8.0+)
CREATE USER 'user'@'%'
  IDENTIFIED WITH 'mysql_native_password' BY 'password'
  REQUIRE NONE
  PASSWORD EXPIRE INTERVAL 30 DAY           -- 30일 후 만료
  ACCOUNT UNLOCK
  PASSWORD HISTORY DEFAULT
  PASSWORD REUSE INTERVAL DEFAULT
  PASSWORD REQUIRE CURRENT DEFAULT;
```

### 2.2 사용자 조회

```sql
-- 모든 사용자 조회
SELECT user, host FROM mysql.user;

-- 특정 사용자만
SELECT user, host, authentication_string FROM mysql.user
WHERE user = 'backup_user';

-- 계정 상태 확인 (Lock 여부)
SELECT user, host, account_locked FROM mysql.user;
```

### 2.3 사용자 삭제

```sql
DROP USER IF EXISTS 'old_user'@'%';
```

## 3. 권한 관리

### 3.1 권한 부여

```sql
-- 전체 데이터베이스 권한
GRANT ALL PRIVILEGES ON *.* TO 'admin'@'%';

-- 특정 데이터베이스
GRANT SELECT, INSERT, UPDATE, DELETE ON dbname.* TO 'app_user'@'10.0.0.%';

-- 읽기 전용 권한
GRANT SELECT ON *.* TO 'readonly'@'%';

-- Replication 권한
GRANT REPLICATION SLAVE ON *.* TO 'repl_user'@'slave_ip';

-- sys 스키마 접근 권한
GRANT SELECT ON sys.* TO 'monitoring'@'%';
```

### 3.2 권한 확인

```sql
-- 특정 사용자의 권한 확인
SHOW GRANTS FOR 'app_user'@'10.0.0.%';

-- 현재 접속 사용자 권한
SHOW GRANTS FOR CURRENT_USER();

-- 데이터베이스별 권한 조회
SELECT user, host, Select_priv, Insert_priv, Update_priv, Delete_priv
FROM mysql.user
WHERE user = 'app_user';
```

### 3.3 권한 제거

```sql
-- 특정 권한 제거
REVOKE SELECT, INSERT ON dbname.* FROM 'app_user'@'10.0.0.%';

-- 모든 권한 제거
REVOKE ALL PRIVILEGES ON *.* FROM 'old_user'@'%';

-- 권한 적용
FLUSH PRIVILEGES;
```

## 4. 세션·트랜잭션 모니터링

### 4.1 현재 연결된 세션 조회

```sql
SHOW FULL PROCESSLIST;
-- 컬럼: Id, User, Host, db, Command, Time, State, Info(SQL)
```

### 4.2 활성 트랜잭션 조회

```sql
-- 진행 중인 트랜잭션 확인
SELECT
  trx_id,
  trx_state,
  trx_started,
  TIMESTAMPDIFF(SECOND, trx_started, NOW()) AS duration_sec,
  trx_mysql_thread_id,
  trx_query
FROM information_schema.innodb_trx
WHERE trx_state = 'RUNNING'
ORDER BY trx_started;
```

### 4.3 Commit 대기 중인 세션

```sql
-- sys 스키마 활용 (간결함)
SELECT
  thd_id,
  conn_id,
  user,
  db,
  command,
  state,
  time,
  current_statement
FROM sys.session
WHERE trx_state = 'ACTIVE'
  AND state = 'waiting for handler commit'
ORDER BY time DESC;
```

### 4.4 오래 수행 중인 쿼리 찾기

```sql
SELECT
  id AS connection_id,
  user,
  host,
  db,
  command,
  time AS running_sec,
  state,
  info AS current_query
FROM information_schema.processlist
WHERE command != 'Sleep'
  AND time > 300  -- 5분 이상
ORDER BY time DESC;
```

### 4.5 특정 유저의 접속 차단

```sql
-- 특정 유저의 모든 세션 강제 종료
SELECT CONCAT('KILL ', id, ';')
FROM information_schema.processlist
WHERE user = 'target_user'
ORDER BY id;

-- 또는 자동 실행
SELECT
  CONCAT('KILL ', id, ';')
INTO OUTFILE '/tmp/kill_sessions.sql'
FROM information_schema.processlist
WHERE user = 'target_user';

SOURCE /tmp/kill_sessions.sql;
```

## 5. General Log 활용

### 5.1 General Log 활성화

```sql
SET GLOBAL general_log='ON';
SET GLOBAL log_output='FILE';  -- 또는 'TABLE'

-- 또는 my.cnf에 설정
[mysqld]
general_log=1
general_log_file=/var/log/mysql/general.log
log_output=FILE
```

### 5.2 실행된 쿼리 내역 조회

```sql
-- 로그 파일에서 특정 쿼리 검색
SELECT * FROM mysql.general_log
WHERE argument LIKE '%UPDATE%'
ORDER BY event_time DESC
LIMIT 20;
```

### 5.3 General Log 비활성화 (성능 영향)

```sql
SET GLOBAL general_log='OFF';
-- General Log는 높은 오버헤드로 트러블슈팅 시에만 활용
```

## 6. Audit Log (감시 및 보안)

### 6.1 Audit Log 활성화

```sql
-- MySQL 8.0 내장 Audit Plugin
INSTALL PLUGIN audit_log SONAME 'audit_log.so';

-- 또는 my.cnf
[mysqld]
plugin-load-add=audit_log.so
audit_log=ON
```

### 6.2 Audit 로그 파일 위치

```sql
SHOW VARIABLES LIKE 'audit_log%';
-- audit_log_file: /var/lib/mysql/audit.log (기본)
```

### 6.3 Audit 이벤트 필터링

```sql
-- DDL 감시
SELECT * FROM mysql.audit_log
WHERE argument LIKE 'ALTER TABLE%'
  OR argument LIKE 'DROP%'
ORDER BY timestamp DESC;

-- 특정 사용자 감시
SELECT * FROM mysql.audit_log
WHERE user = 'admin'
  AND timestamp > DATE_SUB(NOW(), INTERVAL 1 HOUR);
```

## 7. 데이터베이스 및 테이블 정보

### 7.1 데이터베이스 용량 조회

```sql
SELECT
  table_schema,
  ROUND(SUM(data_length + index_length) / 1024 / 1024 / 1024, 2) AS size_gb
FROM information_schema.tables
GROUP BY table_schema
ORDER BY size_gb DESC;
```

### 7.2 테이블별 용량

```sql
SELECT
  table_schema,
  table_name,
  ROUND((data_length + index_length) / 1024 / 1024, 2) AS size_mb,
  table_rows,
  engine
FROM information_schema.tables
WHERE table_schema NOT IN ('mysql', 'information_schema', 'performance_schema')
ORDER BY (data_length + index_length) DESC
LIMIT 50;
```

### 7.3 인덱스 조회

```sql
-- 테이블의 모든 인덱스
SELECT
  table_schema,
  table_name,
  index_name,
  column_name,
  seq_in_index
FROM information_schema.statistics
WHERE table_schema = 'your_db'
  AND table_name = 'your_table'
ORDER BY index_name, seq_in_index;
```

## 8. 마스터 상태 확인 (복제용)

### 8.1 Master 정보

```sql
SHOW MASTER STATUS;
-- 컬럼: File (Binary Log), Position (로그 위치), Binlog_Do_DB, Binlog_Ignore_DB
```

### 8.2 Slave 상태

```sql
SHOW SLAVE STATUS\G
-- 중요 항목:
-- - Slave_IO_Running: 마스터로부터 수신 중?
-- - Slave_SQL_Running: 로컬 DB에 적용 중?
-- - Seconds_Behind_Master: 지연 초 (NULL=정지, 0=동기)
-- - Last_Error: 복제 에러
```

## 11. 유용한 원라이너 쿼리

### 11.1 Kill 세션 자동화

```sql
-- 5분 이상 실행 중인 쿼리 종료
SELECT CONCAT('KILL ', id, ';')
FROM information_schema.processlist
WHERE time > 300 AND command != 'Sleep';
```

### 11.2 Read Only 모드 전환

```sql
-- Slave로 설정 (쓰기 방지)
SET GLOBAL READ_ONLY = ON;

-- 또는 일부 사용자만 쓰기 허용
SET GLOBAL READ_ONLY = ON;
GRANT SUPER ON *.* TO 'admin'@'%';  -- admin은 쓰기 가능
```

### 11.3 Binary Log 강제 로테이션

```sql
FLUSH LOGS;
-- 또는
FLUSH BINARY LOGS;
```

### 11.4 오래된 Binary Log 삭제

```sql
-- 7일 이상 된 로그 삭제
PURGE BINARY LOGS BEFORE DATE_SUB(NOW(), INTERVAL 7 DAY);

-- 또는 특정 파일까지만 유지
PURGE BINARY LOGS TO 'mysql-bin.000050';
```

## 12. 성능 진단 한눈에 보기

```sql
-- MySQL 상태 종합 스냅샷
SELECT
  'Uptime' AS metric, CONCAT(ROUND(VARIABLE_VALUE/3600), ' hours') AS value
FROM information_schema.global_status WHERE VARIABLE_NAME = 'Uptime'
UNION ALL
SELECT 'Max Connections', VARIABLE_VALUE
FROM information_schema.global_variables WHERE VARIABLE_NAME = 'max_connections'
UNION ALL
SELECT 'Active Connections', VARIABLE_VALUE
FROM information_schema.global_status WHERE VARIABLE_NAME = 'Threads_connected'
UNION ALL
SELECT 'Questions', VARIABLE_VALUE
FROM information_schema.global_status WHERE VARIABLE_NAME = 'Questions'
UNION ALL
SELECT 'Slow Queries', VARIABLE_VALUE
FROM information_schema.global_status WHERE VARIABLE_NAME = 'Slow_queries';
```
