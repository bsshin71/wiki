## 1. 핵심 요약

시점 복구(PITR)는 **전체 백업 복구 + 백업 이후의 바이너리 로그 재적용**으로 장애 직전 시점까지 데이터를 되살리는 기법입니다.
핵심은 `xtrabackup_binlog_info`에서 백업 완료 시점의 binlog 파일·position을 확인하고, 그 지점부터 `mysqlbinlog --start-position`으로 SQL을 추출해 순차 적용하는 것입니다.

---

## 2. 복구 원리

> 전체 백업은 "백업 완료 시점"까지만 복구하므로, 그 이후 변경분은 **바이너리 로그**로 보충해야 한다.

1. 전체 백업본으로 복구 (예: 새벽 2시 백업 → `mysql-bin.000126`, position 3925까지)
2. 백업 이후 binlog(3925 ~ 장애직전)를 `mysqlbinlog`로 SQL 변환
3. 변환 SQL을 **데이터 변경 순서대로** 순차 적용

```bash
mysqlbinlog mysql-bin.000126 --start-position=3925 > 000126.sql
mysqlbinlog mysql-bin.000127 > 000127.sql
mysqlbinlog mysql-bin.000129 --stop-position=8564 > 000129.sql
```

> 백업 완료 position은 `<백업디렉토리>/xtrabackup_binlog_info`에서 확인.
> ```
> bin_log.000001  154
> ```

---

## 3. mysqlbinlog 주요 옵션

| 옵션 | 기능 |
|------|------|
| `--start-position=N` | N 이상 position의 첫 이벤트부터 읽기 시작 |
| `--stop-position=N` | N 이상 position의 첫 이벤트에서 중단 (N 직전까지 읽음) |
| `--database=db` | 해당 DB 관련 내용만 추출 |
| `--start-datetime='YYYY-MM-DD HH:MM:SS'` | 지정 시각 이후 첫 이벤트부터 읽기 |
| `--stop-datetime=...` | 지정 시각 직전까지 읽고 중단 |

## 4. 시점 복구 절차 (innobackupex 기준)

```bash
# 1. 현재 상태 확인 → 2. full backup
innobackupex --defaults-file=/etc/my.cnf --no-lock \
  --user=backupuser --password=*** /root/mysqlxtra/fullbak

# 4. DB 정지
systemctl stop mysqld.service

# 5. 복구 준비 (apply-log)
innobackupex --defaults-file=/etc/my.cnf --apply-log /root/mysqlxtra/fullbak/<날짜>

# 6. binlog → SQL 변환 (xtrabackup_binlog_info의 position부터)
mysqlbinlog bin_log.000001 --start-position=154 > 001.sql

# 7. copy-back 복구 (datadir 백업 후)
mv mysql mysql-bak
innobackupex --defaults-file=/etc/my.cnf --copy-back /root/mysqlxtra/fullbak/<날짜>
chown -R mysql:mysql mysql

# 8. DB 시작 → 9. 변환 SQL 적용
systemctl start mysqld.service
cat 001.sql | mysql -uroot -p
```

> 복구 후 테이블 건수가 full backup 이후 변경분까지 포함하면 시점 복구 성공.

## 5. 복구 테스트 검증 (8.0 xtrabackup)

데이터 추가 후 binlog position을 확인하고, 복구 전후 `SHOW MASTER STATUS`로 position 일치를 검증한다.

```sql
SHOW BINARY LOGS;       -- binlog.000010(156), binlog.000011(364)
SHOW MASTER STATUS;     -- binlog.000011, 364
```

```bash
# full backup position 확인
cat ./backup/mysql/fullbackup/2021-02-18/xtrabackup_binlog_info
# binlog.000010  156

# prepare → copy-back
xtrabackup --defaults-file=/etc/my.cnf --prepare --target-dir=.../fullbackup/2021-02-18
xtrabackup --defaults-file=/etc/my.cnf --copy-back --target-dir=.../fullbackup/2021-02-18
chown -R mysql:mysql mysql

# 복구 후 position(156)부터 archive binlog의 변경분 추출·적용
mysqlbinlog ./backup/mysql/archbinlog/arch.binlog.000011 --start-position=156 > mybinlog.sql
cat mybinlog.sql | mysql -u root -p
```

> 적용 후 `SHOW MASTER STATUS` position이 변경분만큼 증가(156→371)하면 검증 완료.

## 6. 오류 상황별 대처

| 증상 | 대처 |
|------|------|
| `os_file_get_status() failed ... Can't determine file permissions` | `setenforce 0` 후 재시작 (SELinux) |
| `Can't connect ... socket '/var/lib/mysql/mysql.sock'` | 중지 → `chmod 777 mysql` → 재시작 |
