## 1. 핵심 요약

상황별 복구 4가지를 정리합니다: ① **단일 테이블만** Transportable Tablespace(DISCARD/IMPORT)로 복구, ② full + **온라인 binlog** 시점 복구, ③ full + **아카이브 binlog** 시점 복구, ④ **실수 DROP** 복구(binlog에서 drop 구문 주석 처리 후 적용).
압축(qpress) 백업은 복구 전 `--decompress`가 선행되어야 합니다.

---

## 2. 사례 1 — 단일 테이블 복구 (Transportable Tablespace)

전체를 복구하지 않고 **특정 테이블(`.ibd`)만** 살릴 때 사용.

```bash
# 1. 압축 백업이면 decompress, 2. 대상 테이블 파일 찾기 (city.ibd)
find ./backup/.../2021-02-24 -name city.*

# 3. prepare + export (→ city.cfg 생성)
xtrabackup --prepare --export --target-dir=.../fullbackup/2021-02-24
```

```sql
-- 4. 대상 테이블의 테이블스페이스 폐기
ALTER TABLE city DISCARD TABLESPACE;
```

```bash
# 5. 백업의 .cfg/.ibd 를 datadir로 복사
cp backup_dir/db/city.cfg data_dir/db
cp backup_dir/db/city.ibd data_dir/db
```

```sql
-- 6. 테이블스페이스 import
ALTER TABLE city IMPORT TABLESPACE;
```

## 3. 사례 2 — Full + 온라인 binlog 시점 복구

```bash
# 1. mysqld stop → 2. datadir 백업/재생성(chown) → 3. decompress
xtrabackup --defaults-file=/etc/my.cnf --user=bkpuser --password=* \
  --decompress --target-dir="full_backup_dir"

# 4. prepare → 5. copy-back → ownership 체크 → 6. start
xtrabackup --prepare  --target-dir="full_backup_dir"
xtrabackup --copy-back --target-dir="full_backup_dir"
chown -R bos:mysql data_dir

# 7. 백업 position 확인
cat .../fullbackup/2021-02-18/xtrabackup_binlog_info   # binlog.000008  156

# 8. 온라인 binlog → SQL 추출 (position부터) → 9. 순차 적용
mysqlbinlog online_binlog_backup_dir/binlog.000008 --start-position=156 > 000008.sql
cat 000008.sql | mysql -u root -p
```

---

## 4. 사례 3 — Full + 아카이브 binlog 시점 복구

사례 2와 동일하나 **8단계에서 archive binlog**를 사용한다.

```bash
mysqlbinlog archive_binlog_dir/arch.binlog.000008 --start-position=156 > 000008.sql
mysqlbinlog archive_binlog_dir/arch.binlog.000009 > 000009.sql
cat 000008.sql | mysql -u root -p
```

---

## 5. 사례 4 — 실수 DROP 복구 (⭐ drop 구문 제외)

복구 SQL에 포함된 `DROP` 구문을 적용하면 다시 삭제되므로 **주석 처리 후 적용**한다.

```bash
# 9. drop 쿼리 위치 확인
grep "drop" 000009.sql        # drop database summary

# 편집기로 해당 라인 주석 처리
vi 000009.sql                 # # drop database summary

# 10. 순차 적용
cat 000008.sql | mysql -u root -p
cat 000009.sql | mysql -u root -p
```

## 6. 복구 테스트 검증 (qpress 압축 백업)

```bash
# qpress 설치 (decompress 전제)
sudo percona-release enable tools
sudo yum install qpress

# mysql stop → datadir 백업/재생성
systemctl stop mysqld
cd /var/lib && sudo mv mysql mysql_backup && sudo mkdir mysql && sudo chown -R mysql:mysql mysql

# 압축 백업 해제 → prepare → copy-back
xtrabackup --defaults-file=/etc/my.cnf --user=bkpuser --password=* \
  --decompress --target-dir=${backupdir}
xtrabackup --prepare  --target-dir=${backupdir}
xtrabackup --copy-back --target-dir=${backupdir}
```

> xtrabackup 실행 계정은 **mysql 그룹**에 포함되어야 하며, 백업 스크립트에 실행 권한이 필요하다.
