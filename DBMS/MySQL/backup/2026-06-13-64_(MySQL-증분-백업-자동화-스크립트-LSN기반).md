## 1. 핵심 요약

전체 백업 후 `xtrabackup_checkpoints`의 `last_lsn`을 **`lastbackupinfo` 파일에 기록**해 두고, 다음 증분 백업이 `--incremental-lsn=${LAST_LSN}`으로 그 지점부터 백업하도록 연결하는 자동화 스크립트입니다.
복구는 full(`--apply-log-only`) → 증분1(`--apply-log-only`) → **마지막 증분(`--apply-log-only` 제외, 롤백 포함)** → copy-back 순서이며, XtraBackup 8.0.22-15 이하의 **PXB-2406 버그**(Unknown error 3611)에 주의합니다.

---

## 2. 백업 구조

| 스크립트 | 역할 |
|----------|------|
| `fullbak4inc.sh` | 최초 전체 백업 + `lastbackupinfo`에 last LSN 기록 |
| `incrementbak.sh` | `lastbackupinfo`의 last LSN 이후 변경분만 증분 백업 |
| `decomp.sh` | 압축(qpress) 백업 해제 |

`lastbackupinfo` 예시:
```
backuppath=/backup/mysql/increment/2021-11-03
backup_type = incremental
from_lsn = 825432863
to_lsn   = 883478947
last_lsn = 883478947
```

---

## 3. Full backup 스크립트 (핵심)

```bash
xtrabackup --defaults-file=${CNFFILE} --backup \
  --user=bkpuser --password=*** --no-server-version-check \
  --socket=/home/bos2/mysql/tmp/mysql.sock --galera-info \
  --parallel=4 --compress --compress-thread=2 \
  --target-dir=${BACKUPDIR} >> ${TRCLOGFILE} 2>&1

# 다음 증분이 참조할 last LSN 기록
echo "backuppath=${BACKUPDIR}" > ${INCBACKUPPATH}/lastbackupinfo
cat ${BACKUPDIR}/xtrabackup_checkpoints >> ${INCBACKUPPATH}/lastbackupinfo
```

## 4. 증분 backup 스크립트 (핵심)

```bash
LAST_LSN=`grep "last_lsn" ${INCBACKUPPATH}/lastbackupinfo | cut -d'=' -f2 | tr -d ' '`

xtrabackup --defaults-file=${CNFFILE} --backup \
  --user=bkpuser --password=*** --no-server-version-check \
  --socket=... --galera-info --parallel=4 --compress --compress-thread=2 \
  --incremental-lsn=${LAST_LSN} \        # ← 이전 백업 last LSN 이후부터
  --target-dir=${BACKUPDIR} >> ${TRCLOGFILE} 2>&1

# 다음 증분용 last LSN 갱신
echo "backuppath=${BACKUPDIR}" > ${INCBACKUPPATH}/lastbackupinfo
cat ${BACKUPDIR}/xtrabackup_checkpoints >> ${INCBACKUPPATH}/lastbackupinfo
```

> 운영은 일 1회 증분이라 overwrite 없으나, 동일 경로 충돌 방지를 위해 증분본을 `-N1`, `-N2`로 rename 후 누적 보관.

## 5. 복구 과정

```bash
# 1) 압축 해제 (decomp.sh)

# 2) 전체 백업 prepare (apply-log-only)
xtrabackup --apply-log-only --prepare --target-dir=/backup/mysql/fullbackup/2021-11-03

# 3) 1차 증분 적용 (apply-log-only)
xtrabackup --apply-log-only --prepare \
  --target-dir=/backup/mysql/fullbackup/2021-11-03 \
  --incremental-dir=/backup/mysql/increment/2021-11-03-N1

# 4) 마지막 증분 적용 (⭐ --apply-log-only 제외 → 롤백 포함)
xtrabackup --prepare \
  --target-dir=/backup/mysql/fullbackup/2021-11-03 \
  --incremental-dir=/backup/mysql/increment/2021-11-03

# 5) DB 중지 → 6) copy-back → 7) start
xtrabackup --copy-back --target-dir=/backup/mysql/fullbackup/2021-11-03
```

> 마지막 증분만 `--apply-log-only`를 빼는 이유: 완료되지 않은 트랜잭션을 롤백해야 하기 때문.

## 6. ⚠️ PXB-2406 버그 (복구 오류)

XtraBackup **8.0.22-15 이하**에서 prepare 시 발생 가능:
```
xtrabackup: Unknown error 3611
... handle_fatal_signal ... dd_table_open_on_name ...
```
- 원인: Percona XtraBackup 버그 (PXB-2406).
- 조치: **XtraBackup을 8.0.23-16 이상으로 업그레이드** 후 재시도.
