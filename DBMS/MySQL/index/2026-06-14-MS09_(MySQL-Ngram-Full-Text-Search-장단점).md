## 1. 핵심 요약

N-gram FULLTEXT는 텍스트를 **연속 n글자 묶음**으로 잘라 역색인을 만들어, 띄어쓰기 없는 **CJK(한/중/일)** 와 `LIKE '%키워드%'` 대체에 유효하다. 하지만 **인덱스 비대화(가장 큰 단점)**, 쓰기 비용 증가, 노이즈·랭킹 품질 한계가 있어, 대용량·고품질 검색 요구에는 **전용 검색엔진 병행**이 유리하다.

---

## 2. 장점

- CJK 등 토큰화 어려운 언어에서 동작(공백 없이 붙은 문자열 검색 가능).
- `LIKE '%...%'` 풀스캔 대비 역색인 기반 → 긴 텍스트·다건일수록 빠름.
- 외부 검색엔진 없이 **MySQL 단독**으로 그럴듯한 검색 제공.

## 3. 단점

| 단점 | 내용 |
|------|------|
| **인덱스 비대화** | 길이 L 문자열 → 약 (L−n+1) 토큰. 디스크↑·백업/복구↑·버퍼풀 압박↑ |
| 쓰기 비용↑ | 행 변경마다 다수 ngram 토큰 갱신 → 쓰기 성능 저하, 대량적재 재구축 부담 |
| 정확도/랭킹 | 문자 조각 단위라 흔한 2-gram 노이즈, 짧은 검색어·숫자/코드 혼재 시 부정확 |
| 설정 제약 | `ngram_token_size` 변경 시 인덱스 재구축, 일반 FTS 최소/최대 단어길이 미적용 |
| 고급기능 한계 | 동의어·형태소·오타교정·하이라이트·필드부스팅 제한 |

## 4. n 값·적용 판단

- **n=1**: 토큰·노이즈 폭발로 비추천. **CJK는 보통 2~3**(정확도 vs 크기/부하 타협).
- 읽기 위주 + 텍스트 길이 통제 가능 + 단순 키워드검색 → ngram FTS 적합.
- 수천만~억 건 + 쓰기 많음 + 검색품질 요구 높음 → **전용 검색엔진 병행** 유리.
