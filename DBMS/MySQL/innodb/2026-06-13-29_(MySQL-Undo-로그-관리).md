## 1. 핵심 요약

Undo 로그는 트랜잭션 롤백과 MVCC를 위해 변경 전 데이터를 보관합니다.
MySQL 8.0에서는 undo를 독립 테이블스페이스로 관리하며, 과도하게 커진 undo는 **자동(`innodb_undo_log_truncate`)** 또는 **수동(INACTIVE→ACTIVE)** 모드로 공간을 회수합니다.

---

## 2. Undo 사용량 조회

```sql
-- 활성 트랜잭션의 undo 레코드 수 (history length)
SELECT count FROM information_schema.innodb_metrics
WHERE SUBSYSTEM='transaction' AND NAME='trx_rseg_history_len';
```

```sql
-- undo 로그가 사용하는 파일 조회
SELECT tablespace_name, file_name
FROM information_schema.FILES
WHERE file_type LIKE 'UNDO LOG';
-- innodb_undo_001 | ./undo_001
-- innodb_undo_002 | ./undo_002
```

---

## 3. Undo 테이블스페이스 추가

```sql
-- 절대경로 지정
CREATE UNDO TABLESPACE extra_undo_003 ADD DATAFILE '/home/bos2/mysql/data/undo_003.ibu';

-- 상대경로(data dir 기준)
CREATE UNDO TABLESPACE extra_undo_004 ADD DATAFILE 'undo_004.ibu';
```

---

## 4. Undo 테이블스페이스 삭제

```sql
-- 바로 삭제 불가
DROP UNDO TABLESPACE `extra_undo_004`;
-- ERROR 1529: Failed to drop UNDO TABLESPACE extra_undo_004

-- INACTIVE 로 변경 후 삭제
ALTER UNDO TABLESPACE extra_undo_004 SET INACTIVE;
DROP UNDO TABLESPACE `extra_undo_004`;
```

---

## 5. Undo 공간 확보 (truncate)

언두 파일이 과도하게 커진 경우 아래 두 모드 중 하나로 회수.

### 5.1 자동 모드

```sql
-- innodb_undo_log_truncate = ON 설정
SHOW VARIABLES LIKE 'innodb_purge%';
-- innodb_purge_batch_size              | 300
-- innodb_purge_rseg_truncate_frequency | 128   (작을수록 빈번하게 truncate)
-- innodb_purge_threads                 | 4
```

### 5.2 수동 모드

```sql
-- 비활성화 → purge 스레드가 공간 반납 → 다시 활성화
ALTER UNDO TABLESPACE innodb_undo_001 SET INACTIVE;
ALTER UNDO TABLESPACE innodb_undo_001 SET ACTIVE;
```
