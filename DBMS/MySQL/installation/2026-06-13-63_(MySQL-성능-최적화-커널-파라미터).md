## 1. 핵심 요약

MySQL 서버의 동시 접속·대용량 트랜잭션 처리를 위해 Linux 커널 파라미터(`/etc/sysctl.conf`)와 파일 디스크립터 한도(`/etc/security/limits.conf`)를 조정합니다.
핵심은 **파일 디스크립터 증가, swappiness/dirty 비율 축소, 네트워크 백로그 확대, AIO 한도 증가**입니다.

---

## 2. 파일 디스크립터

```bash
# /etc/sysctl.conf
fs.file-max = 100000          # 시스템 전체 최대 오픈 파일 수
```
```bash
# /etc/security/limits.conf — MySQL 프로세스 한도
mysql soft nofile 65535
mysql hard nofile 65535
```

---

## 3. 메모리 / 공유 메모리

```bash
vm.swappiness = 10            # 스왑 최소화 (메모리 충분 시 0~10)
vm.dirty_ratio = 15           # RAM에 유지 가능한 dirty page 비율
vm.dirty_background_ratio = 5 # 백그라운드 flush 시작 비율
vm.overcommit_memory = 1      # 오버커밋 허용 (InnoDB 메모리 자유도)
```

---

## 4. 네트워크

```bash
net.core.somaxconn = 4096          # 동시 접속 요청 큐
net.core.netdev_max_backlog = 5000 # NIC 큐 최대 패킷
net.ipv4.tcp_max_syn_backlog = 8192# SYN 큐 크기
net.ipv4.tcp_fin_timeout = 15      # TIME_WAIT 정리 가속
net.ipv4.tcp_tw_reuse = 1          # TIME_WAIT 소켓 재사용
```

---

## 5. InnoDB (비동기 I/O)

```bash
fs.aio-max-nr = 1048576       # 비동기 I/O 요청 최대치 (높은 동시성)
```

---

## 6. 적용 방법

```bash
# 커널 파라미터 적용
sudo sysctl -p
sudo sysctl -a                # 전체 확인

# 파일 디스크립터: limits.conf 추가 후
# pam_limits.so 가 /etc/pam.d/common-session(-noninteractive)에 포함됐는지 확인
```

---

## 7. 적용 사례 (cfddb1 기준)

| 파라미터 | default | 권장 |
|----------|---------|------|
| fs.file-max | 1596600 | 100000 |
| vm.swappiness | 30 | 10 |
| vm.dirty_ratio | 30 | 15 |
| vm.dirty_background_ratio | 10 | 5 |
| net.core.somaxconn | 2048 | 4096 |
| net.core.netdev_max_backlog | 1000 | 5000 |
| net.ipv4.tcp_max_syn_backlog | 512 | 8192 |
| net.ipv4.tcp_fin_timeout | 60 | 15 |
| net.ipv4.tcp_tw_reuse | 2 | 1 |
| fs.aio-max-nr | 65536 | 1048576 |

> `fs.file-max`는 기본값(159만)이 이미 충분히 커서 굳이 낮출 필요는 없음.
