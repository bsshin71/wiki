## 1. 핵심 요약

MySQL 8.0+에서 락 경합 시 **누가(blocking) 누구를(waiting) 막는지**는 `performance_schema.data_lock_waits` + `threads` + `events_statements_current` 조인이나 `sys.schema_table_lock_waits` 뷰로 한 번에 확인합니다.
가장 간단한 진단은 `sys.schema_table_lock_waits`(KILL 가이드 컬럼 포함)이며, 조치는 `KILL <blocking_pid>`입니다.

---

## 2. 블로킹 세션 조회 (Performance Schema)

```sql
SELECT
  dlw.OBJECT_SCHEMA       AS waiting_schema,
  dlw.OBJECT_NAME         AS waiting_table,
  t_wait.PROCESSLIST_ID   AS waiting_pid,
  es_wait.SQL_TEXT        AS waiting_sql,
  t_block.PROCESSLIST_ID  AS blocking_pid,
  es_block.SQL_TEXT       AS blocking_sql,
  dlw.LOCK_MODE           AS waiting_lock_mode,
  dlb.LOCK_MODE           AS blocking_lock_mode
FROM performance_schema.data_lock_waits w
JOIN performance_schema.data_locks dlw ON dlw.engine_lock_id = w.requesting_engine_lock_id
JOIN performance_schema.data_locks dlb ON dlb.engine_lock_id = w.blocking_engine_lock_id
JOIN performance_schema.threads  t_wait  ON t_wait.thread_id  = dlw.thread_id
JOIN performance_schema.threads  t_block ON t_block.thread_id = dlb.thread_id
LEFT JOIN performance_schema.events_statements_current es_wait  ON es_wait.THREAD_ID  = t_wait.thread_id
LEFT JOIN performance_schema.events_statements_current es_block ON es_block.THREAD_ID = t_block.thread_id
ORDER BY blocking_pid, waiting_pid;
```

> blocking 트랜잭션의 지속 시간을 보려면 `information_schema.innodb_trx`의 `trx_started`를 조인해 `TIMESTAMPDIFF(SECOND, trx_started, NOW())`로 계산.

---

## 3. sys.schema_table_lock_waits (권장·간단)

```sql
SELECT *
FROM sys.schema_table_lock_waits
-- WHERE object_schema = 'your_db' AND object_name = 'your_table'
ORDER BY waiting_pid;
```

- `waiting_pid`, `blocking_pid`, `waiting_query`, `blocking_query`와
- **`sql_kill_blocking_connection`**(바로 KILL 가능한 가이드 컬럼) 포함.

## 4. 특정 테이블을 잡고 있는 세션 나열

```sql
SELECT t.PROCESSLIST_ID AS pid, t.PROCESSLIST_USER AS user,
       es.SQL_TEXT AS current_sql
FROM performance_schema.data_locks l
JOIN performance_schema.threads t ON t.thread_id = l.thread_id
LEFT JOIN performance_schema.events_statements_current es ON es.thread_id = l.thread_id
WHERE l.OBJECT_SCHEMA = 'your_db' AND l.OBJECT_NAME = 'your_table'
ORDER BY pid;
```

> SQL_TEXT가 비면 컨슈머 활성화:
> ```sql
> UPDATE performance_schema.setup_consumers SET ENABLED='YES'
>  WHERE NAME IN ('events_statements_current','events_statements_history','events_statements_history_long');
> ```

## 5. 빠른 조치

1. 막고 있는 쿼리 확인 → `sys.schema_table_lock_waits` (가장 간단)
2. 세션 종료 → `KILL <blocking_pid>`
3. 전체 락/대기 스냅샷 → `SHOW ENGINE INNODB STATUS\G`
