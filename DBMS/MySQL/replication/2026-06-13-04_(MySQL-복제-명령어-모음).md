## 1. 핵심 요약

MySQL 복제 운영 시 자주 사용하는 SQL 명령어를 정리한 가이드입니다.
Master/Slave 구분, 상태 확인, 초기화, 채널 관리 등의 명령어를 포함합니다.

---

## 2. Master 상태 확인

### 2.1 Master/Slave 구분

```sql
SHOW VARIABLES LIKE 'read_only';
-- read_only = ON  → Slave (읽기 전용)
-- read_only = OFF → Master (쓰기 허용)
```

### 2.2 Master 정보 조회

```sql
SHOW MASTER STATUS;

-- 출력 항목:
-- File: mysql-bin.000001           (현재 Binary Log 파일명)
-- Position: 155                     (파일 내 위치)
-- Binlog_Do_DB: test               (복제 대상 DB)
-- Binlog_Ignore_DB: (없음)         (복제 제외 DB)
-- Executed_Gtid_Set: (GTID 목록)   (GTID 기반 복제 시)
```

### 2.3 Master에 연결된 Slave 목록

```sql
SHOW SLAVE HOSTS;

-- 출력:
-- Server_id: 2,3
-- Host: (IP 또는 빈 값)
-- Port: 3306
-- Master_id: 1
-- Slave_UUID: (UUID)
```

## 3. Slave 상태 관리

### 3.1 Slave 시작/중단

```sql
-- Slave 복제 시작
START SLAVE;

-- 또는 (MySQL 8.0.22+)
START REPLICA;

-- Slave 복제 중단
STOP SLAVE;

-- 또는
STOP REPLICA;
```

### 3.2 특정 Thread만 제어

```sql
-- I/O Thread만 중단 (Master로부터 읽기 중단)
STOP SLAVE IO_THREAD;

-- SQL Thread만 중단 (Relay Log 재생 중단)
STOP SLAVE SQL_THREAD;

-- 재시작
START SLAVE IO_THREAD;
START SLAVE SQL_THREAD;
```

---

## 4. Slave 상태 조회

### 4.1 전체 복제 상태

```sql
SHOW SLAVE STATUS\G

-- 주요 항목:
-- Slave_IO_Running: Yes/No         (I/O Thread 상태)
-- Slave_SQL_Running: Yes/No        (SQL Thread 상태)
-- Seconds_Behind_Master: 0         (지연 시간, 0=동기)
-- Last_Error: (에러 메시지)        (복제 에러)
-- Master_Log_File: mysql-bin.000001 (Master 로그 파일)
-- Read_Master_Log_Pos: 1234        (Master 읽은 위치)
-- Relay_Log_File: relay-bin.000001 (Slave Relay Log)
-- Relay_Log_Pos: 5678              (Slave 재생 위치)
```

### 4.2 간단 상태 확인

```sql
SHOW SLAVE STATUS;  -- 테이블 형식 (더 간결)
```

## 5. Master 채널 설정

### 5.1 Position 기반 Replication

```sql
CHANGE MASTER TO
  MASTER_HOST='master_ip',
  MASTER_PORT=3306,
  MASTER_USER='repl_user',
  MASTER_PASSWORD='password',
  MASTER_LOG_FILE='mysql-bin.000001',
  MASTER_LOG_POS=1234;
```

### 5.2 GTID 기반 Replication

```sql
CHANGE MASTER TO
  MASTER_HOST='master_ip',
  MASTER_PORT=3306,
  MASTER_USER='repl_user',
  MASTER_PASSWORD='password',
  MASTER_AUTO_POSITION=1;
```

### 5.3 Multi-Source Replication (채널별)

```sql
CHANGE MASTER TO
  MASTER_HOST='master1_ip',
  MASTER_USER='repl_user',
  MASTER_PASSWORD='password',
  FOR CHANNEL 'master1';

CHANGE MASTER TO
  MASTER_HOST='master2_ip',
  MASTER_USER='repl_user',
  MASTER_PASSWORD='password',
  FOR CHANNEL 'master2';

-- 채널별 시작
START SLAVE FOR CHANNEL 'master1';
START SLAVE FOR CHANNEL 'master2';
```

## 6. Slave 초기화

### 6.1 부분 초기화 (Position 정보만 삭제)

```sql
RESET SLAVE;  -- Slave 복제 정보 제거, 데이터는 유지
```

### 6.2 전체 초기화 (Master 채널까지 삭제)

```sql
RESET SLAVE ALL;  -- Master 채널 정보 + GTID 정보 전부 삭제
```

---

## 7. Master Binary Log 관리

### 7.1 Binary Log 강제 로테이션

```sql
FLUSH LOGS;  -- 현재 Binary Log 종료, 새 파일 시작
-- 또는
FLUSH BINARY LOGS;
```

### 7.2 오래된 Binary Log 삭제

```sql
-- 7일 이상 된 로그 삭제
PURGE BINARY LOGS BEFORE DATE_SUB(NOW(), INTERVAL 7 DAY);

-- 특정 파일까지만 유지
PURGE BINARY LOGS TO 'mysql-bin.000050';

-- 모든 로그 삭제 (⚠️ 위험!)
RESET MASTER;
```

## 8. GTID 관련 명령어

### 8.1 GTID 상태 확인

```sql
SHOW VARIABLES LIKE 'gtid%';

-- 주요 변수:
-- gtid_mode: ON/OFF
-- enforce_gtid_consistency: ON/OFF
-- gtid_executed: (실행된 GTID SET)
-- gtid_purged: (Binary Log에 없는 GTID SET)
```

### 8.2 특정 GTID 건너뛰기

```sql
SET SESSION gtid_next='3E11FA47-71CA-11E1-9E33-C80AA9429562:100';
BEGIN; COMMIT;
SET SESSION gtid_next='AUTOMATIC';
```

### 8.3 GTID 초기화

```sql
RESET MASTER;  -- Master에서 모든 GTID 제거

RESET SLAVE ALL;  -- Slave에서 GTID 정보 제거
```

## 9. Read-Only 제어

### 9.1 Slave를 읽기 전용으로 설정

```sql
-- Slave에서 실행
SET GLOBAL READ_ONLY=ON;

-- 슈퍼유저(admin)는 여전히 쓰기 가능
-- 일반 사용자는 읽기만 가능
```

### 9.2 Read-Only 해제

```sql
SET GLOBAL READ_ONLY=OFF;
```

## 10. 장애 복구 명령어

### 10.1 복제 에러 건너뛰기

```sql
-- Position 기반: 한 건 건너뛰기
SET GLOBAL SQL_SLAVE_SKIP_COUNTER=1;
START SLAVE;

-- GTID 기반: GTID 건너뛰기 (위 섹션 참고)
```

### 10.2 Slave 동기화 재설정

```sql
-- Slave 중단
STOP SLAVE;

-- 복제 정보 초기화
RESET SLAVE ALL;

-- Master 채널 재설정 (위 섹션 참고)
CHANGE MASTER TO ...;

-- 복제 재시작
START SLAVE;
```

## 11. 성능 튜닝 명령어

### 11.1 병렬 복제 설정

```sql
SET GLOBAL slave_parallel_workers=4;      -- 4개 스레드
SET GLOBAL slave_parallel_type='LOGICAL_CLOCK';  -- GTID 기반 병렬화

STOP SLAVE;
START SLAVE;
```

### 11.2 복제 지연 모니터링

```sql
SELECT 
  VARIABLE_VALUE AS seconds_behind_master 
FROM INFORMATION_SCHEMA.PROCESSLIST 
WHERE COMMAND='Binlog Dump';

-- 또는
SHOW SLAVE STATUS\G  -- Seconds_Behind_Master 항목 확인
```

## 12. 자주 사용하는 명령어 체크리스트

| 상황 | 명령어 |
|------|--------|
| Slave 상태 확인 | `SHOW SLAVE STATUS\G` |
| Master 상태 확인 | `SHOW MASTER STATUS` |
| Slave 시작 | `START SLAVE` |
| Slave 중단 | `STOP SLAVE` |
| Master 채널 설정 | `CHANGE MASTER TO ...` |
| 복제 에러 건너뛰기 | `SET GLOBAL SQL_SLAVE_SKIP_COUNTER=1` |
| Slave 초기화 | `RESET SLAVE ALL` |
| Read-Only 설정 | `SET GLOBAL READ_ONLY=ON` |
