## 1. 핵심 요약

여러 소스 서버(stdb1, stdb2)를 한 대의 replica(stdb3)로 모으는 구성으로, **채널(channel)별 `CHANGE MASTER TO ... FOR CHANNEL`**로 설정합니다.
초기 데이터는 큰 DB는 xtrabackup, 작은 DB는 mysqldump로 혼용 이관하고, 각 소스의 GTID를 합쳐 `gtid_purged`에 설정합니다.
⚠️ **멀티 소스 채널 복제에서는 semi-sync 복제를 사용할 수 없습니다.**

---

## 2. 초기 데이터 구축 전략

- 소스 2대 이상 → 각각 다른 방식으로 이관(충돌 방지).
- 큰 DB: **xtrabackup**, 작은 DB: **mysqldump** 혼용.
- 전부 큰 DB면 xtrabackup + ibd export/import 혼용.

---

## 3. 구축 과정 (stdb1·stdb2 → stdb3)

### 3.1 stdb1 복제 (xtrabackup)
- stdb1을 xtrabackup으로 백업 후 stdb3에 복제 (gtid based slave 추가 절차 참고).

### 3.2 stdb2 복제 (mysqldump)
```sql
-- read_only 전환 후 gtid_executed 기록
SHOW VARIABLES LIKE '%gtid%';
-- gtid_executed: 2d511b4d-...:1-60231
```
```bash
mysqldump -uroot -pxxxx --set-gtid-purged=OFF --routines --single-transaction BOSPR > bospr.sql
```
- dump를 stdb3에 import.

### 3.3 stdb3 구성 (GTID 합산 + 채널 연결)
```sql
RESET MASTER;
SET GLOBAL gtid_purged =
  "08b0fb1c-...:1-1628472,2d511b4d-...:1-60231";   -- 두 소스 GTID 합산
```
> stdb1 GTID는 `xtrabackup_binlog_info`, stdb2 GTID는 mysqldump 전 기록값 사용.

```sql
CHANGE MASTER TO MASTER_HOST="10.11.14.141", MASTER_USER="repl",
  MASTER_PASSWORD="xxxx", MASTER_AUTO_POSITION=1 FOR CHANNEL 'stdb1';
CHANGE MASTER TO MASTER_HOST="10.11.14.142", MASTER_USER="repl",
  MASTER_PASSWORD="xxxx", MASTER_AUTO_POSITION=1 FOR CHANNEL 'stdb2';

START SLAVE FOR CHANNEL 'stdb1';
START SLAVE FOR CHANNEL 'stdb2';
SHOW SLAVE STATUS\G   -- 채널별로 Slave_IO/SQL_Running=Yes 확인
```

## 4. 주의점 — semi-sync 불가

> Multi-source channel replication에서는 **semi-sync replication을 사용할 수 없음** (MySQL Bug #102994).

증상(semi-sync 혼용 시):
```
[ERROR] [MY-010557] Error reading packet from server for channel 'stdb2':
  Lost connection to MySQL server during query (server_errno=2013)
Last_IO_Errno: 13122  Relay log write failure: could not queue event from master
```
