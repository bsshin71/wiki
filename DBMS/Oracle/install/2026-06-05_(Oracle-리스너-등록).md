## 1. 핵심 요약
- Oracle 리스너는 **기본적으로 수동 등록 불필요**. DB 기동 시 LREG 프로세스가 자동으로 등록한다.
- `listener.ora` 정적 등록은 **"DB가 꺼진 상태에서 원격 제어"가 필요할 때만** 사용한다.

## 2. 상세 설명

### 동적 등록 (Dynamic Registration) — 기본 방식

DB 인스턴스가 `STARTUP`되면 **LREG** 백그라운드 프로세스가 자동으로 동작한다:

1. 1521 포트로 리스너 생존 여부 확인
2. 리스너에게 DB 이름·상태 자동 주입
3. DB 기동 후 **약 30초~1분 이내** 자동 완료

→ `listener.ora` 를 수정하지 않아도 외부 접속이 가능하다.

---

### 정적 등록 (Static Registration) — 예외 상황

**사용 목적**: DB가 `SHUTDOWN` 상태일 때도 원격 클라이언트에서 접속해 DB를 기동해야 하는 경우.

동적 등록은 DB가 꺼지면 리스너가 해당 DB를 목록에서 제거하므로, 원격으로 `sysdba` 접속이 불가능해진다.

**`listener.ora` 수정 예시** (`$TNS_ADMIN/listener.ora`):

```
LISTENER =
  (DESCRIPTION_LIST =
    (DESCRIPTION =
      (ADDRESS = (PROTOCOL = TCP)(HOST = Rocky9.7)(PORT = 1521))
      (ADDRESS = (PROTOCOL = IPC)(KEY = EXTPROC1521))
    )
  )

# 정적 등록 — DB가 꺼져 있어도 강제 중계
SID_LIST_LISTENER =
  (SID_LIST =
    (SID_DESC =
      (GLOBAL_DBNAME = devdb)
      (ORACLE_HOME = /oracle/u01/app/oracle/product/19.3.0/dbhome_1)
      (SID_NAME = devdb)
    )
    (SID_DESC =
      (GLOBAL_DBNAME = proddb)
      (ORACLE_HOME = /oracle/u01/app/oracle/product/19.3.0/dbhome_1)
      (SID_NAME = proddb)
    )
  )
```

수정 후 반영:
```bash
lsnrctl reload
```

---

### 동적 vs 정적 등록 비교

| 항목 | 동적 등록 | 정적 등록 |
|------|-----------|-----------|
| `listener.ora` 수정 | 불필요 | 필요 |
| 등록 조건 | DB 기동 시 자동 | 항상 고정 |
| DB 꺼진 상태 원격 접속 | ❌ 불가 | ✅ 가능 |
| `lsnrctl status` 상태 | `READY` | `UNKNOWN` |
| 권장 환경 | 일반 개발·운영 | 원격 DBA 제어 필요 시 |

> `UNKNOWN` 상태는 오류가 아니라 정적 등록의 정상 상태다.
