## 1. 핵심 요약
- Oracle 19c에서 사용자 계정을 생성하려면 **sysdba 접속 → `_ORACLE_SCRIPT` 활성화 → 계정 생성 → 권한 부여** 4단계가 필요하다.

## 2. 상세 설명

### 1단계 — 관리자 접속

```sql
sqlplus "/as sysdba"
```

### 2단계 — C## 제한 우회 (Oracle 12c 이상 필수)

Oracle 12c부터 멀티테넌트 아키텍처로 인해 계정명 앞에 `C##`을 붙여야 하는 제한이 생겼다.
기존 방식(C## 없이)으로 계정을 만들려면 세션 설정을 먼저 변경해야 한다.

```sql
ALTER SESSION SET "_ORACLE_SCRIPT" = TRUE;
```

> ⚠️ 이 단계를 빠뜨리면 **ORA-65096** 오류가 발생한다.

### 3단계 — 계정 생성

```sql
CREATE USER 사용자명 IDENTIFIED BY 비밀번호;
```

> 비밀번호에 특수문자가 포함되면 쌍따옴표(`""`)로 감싼다.

### 4단계 — 접속 및 자원 권한 부여

```sql
GRANT CONNECT, RESOURCE TO 사용자명;
```

계정 생성 직후에는 아무 권한이 없어 접속조차 불가하므로 반드시 필요하다.

---

### 선택 설정

| 목적 | 명령어 |
|------|--------|
| 테이블스페이스 쿼터 무제한 | `ALTER USER 사용자명 DEFAULT TABLESPACE USERS QUOTA UNLIMITED ON USERS;` |
| DBA 전체 권한 부여 (테스트용) | `GRANT DBA TO 사용자명;` |

> DBA 권한은 운영 환경에서는 사용하지 않는다.

---

### 에러 대처

| 에러 코드 | 원인 | 해결 |
|-----------|------|------|
| ORA-65096 | `_ORACLE_SCRIPT` 설정 누락 | 2단계 `ALTER SESSION` 먼저 실행 |
