## 1. 핵심 요약

MySQL 실행계획의 품질은 **통계정보(innodb_stats)와 히스토그램** 정확도에 달려 있다. `innodb_stats_persistent=ON`으로 영구 저장하고, `ANALYZE TABLE`로 수동 갱신, `UPDATE HISTOGRAM`으로 칼럼별 분포 수집. slow query는 pt-query-digest로 분석한다.

---

## 2. 통계정보 설정

```sql
-- 테이블별 영구 통계 저장
SHOW VARIABLES LIKE '%stats_persistent';  -- innodb_stats_persistent: ON (기본)
ALTER TABLE employees STATS_PERSISTENT=1;

-- 인덱스 통계 조회
SELECT * FROM mysql.innodb_index_stats
WHERE database_name='employees' AND table_name='employees';

-- 통계 수동 갱신
ANALYZE TABLE employees.employees;
```

### 자동갱신 설정

| 변수 | 기본 | 동작 |
|------|------|------|
| `innodb_stats_auto_recalc` | ON | 전체 DB 통계 자동 갱신 |
| `STATS_AUTO_RECALC=1` | — | 테이블 자동 갱신 |
| `STATS_AUTO_RECALC=0` | — | `ANALYZE TABLE` 시에만 갱신 |

## 3. 히스토그램

```sql
-- 수집 (자동 미수집, 수동만)
ANALYZE TABLE employees.employees UPDATE HISTOGRAM ON gender, hire_date;

-- 조회
SELECT * FROM information_schema.column_statistics
WHERE schema_name='employees' AND table_name='employees'\G;

-- 삭제
ANALYZE TABLE employees.employees DROP HISTOGRAM ON gender;
```

## 4. Slow Query 분석

```bash
# pt-query-digest로 slow log 분석
pt-query-digest --type='slowlog' mysql-query.log > slow_stat.log
```
