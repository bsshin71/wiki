## 1. 핵심 요약

실무 MySQL 튜닝 패턴 2가지: ① **지연된 조인(Deferred Join)** — GROUP BY+LIMIT 처리 후 외부 테이블과 조인해 조인 행 수 대폭 감소. ② **래터럴 조인(Lateral Join)** — 서브쿼리에서 외부 칼럼 참조 가능, 행별 소량 조회에 유용하나 Rematerialize 비용 주의.

---

## 2. 지연된 조인

### 원본 쿼리 (비효율)
```sql
-- salaries·employees 전체 조인 후 GROUP BY·LIMIT
SELECT e.* FROM salaries s, employees e
WHERE e.emp_no = s.emp_no AND s.emp_no BETWEEN 10001 AND 13000
GROUP BY s.emp_no ORDER BY SUM(s.salary) DESC LIMIT 10;
-- 문제: 28,606건 조인 → GROUP BY → LIMIT
```

### 튜닝 쿼리 (지연된 조인)
```sql
-- salaries에서 먼저 GROUP BY·ORDER BY·LIMIT → 10건만 employees와 조인
SELECT e.* FROM
  (SELECT s.emp_no FROM salaries s
   WHERE s.emp_no BETWEEN 10001 AND 13000
   GROUP BY s.emp_no ORDER BY SUM(s.salary) DESC LIMIT 10) x,
  employees e
WHERE e.emp_no = x.emp_no;
-- 효과: 10건만 employees와 eq_ref 조인
```

> ✅ **핵심**: LIMIT 전에 조인하지 않고, **서브쿼리로 필요한 key(10건)만 먼저 추출** 후 조인.

## 3. 래터럴 조인 (LATERAL)

```sql
-- Matt 사원들의 최근 급여 변경 2건씩 조회
SELECT * FROM employees e
LEFT JOIN LATERAL (
  SELECT * FROM salaries s
  WHERE s.emp_no = e.emp_no
  ORDER BY s.from_date DESC LIMIT 2
) s2 ON s2.emp_no = e.emp_no
WHERE e.first_name = 'Matt';
```
- **외부 칼럼(`e.emp_no`) 참조** 가 가능 → 서브쿼리가 행별로 독립 실행(for-each 효과).
- 외부 행 수만큼 **Materialize(임시테이블) 반복** → 외부 행이 많으면 성능 저하.
- `FROM` 절에서도 LATERAL 사용 시 외부 칼럼 참조 가능.
