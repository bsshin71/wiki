## 1. 핵심 요약

Obsidian + Claude Code(Claudian) 기반 개인 학습 지식 관리 시스템(LLM Wiki) 구축 기록.
Obsidian 설치 → 플러그인 설치 → VS Code에서 구축 작업 → 운영 중 점진적 개선 순서로 진행한다.
token 사용량 절감과 max request limit 우회를 위해 graphify·run-batch·ingest·lint 등 커스텀 Skill·Command를 활용한다.

---

## 2. 구축 과정

### 1단계 — Obsidian 설치

- 설치 과정은 인터넷 자료 참고

### 2단계 — Obsidian 플러그인 설치

| 플러그인 | 용도 |
|---------|------|
| **Templater** | 신규 문서 작성 시 template 자동 생성 |
| **Claudian** | 구축 후 운영에 사용할 Claude Code 전용 터미널 |
| **Git** | GitHub에 생성 문서를 업로드할 때 사용. GitHub MCP보다 편함 |
| **Calendar** | Daily note 생성용 |
| **Markdown Formatting Assistant** | Markdown 에디터 helper |

### 3단계 — VS Code에서 구축 작업 진행

- 숨김 폴더 및 숨김 파일(`.claude`, `.obsidian` 등)을 보면서 작업해야 하는데 VS Code에서 가능함
- **구축**: VS Code + VS Code Extension(Claude Code)에서 작업
- **운영**: Obsidian + Obsidian의 Claude Code 플러그인인 Claudian에서 수행
- 배치 파일이나 Python 프로그램 작성 시 VS Code가 유리함

### 4단계 — 시행착오 필수

- raw 문서 → 최종 문서로 가공은 LLM(Claude Code)에 의해 처리되는데 이 과정에서 적지 않은 시행착오가 있다
- **1주일 정도 운영 과정을 거치면서 점진적으로 프로세스를 개선**한다

## 3. 디렉토리 구조

```
LLMWiki
├── CLAUDE.md                  ← Claude Code 행동 지침
├── .schema/
│   ├── system_prompt.md       ← AI 상세 행동 지침
│   └── template_note.md       ← 문서 템플릿
├── raw/                       ← 원본 자료 (★ 절대 이동·삭제 금지, 최초 위치 유지)
│   │
│   │  [ingest 스캔 제외 — 2개 폴더]
│   ├── attachments/           ← Obsidian 자동 저장 첨부파일·이미지
│   ├── pdf2md/                ← PDF 변환 산출물 (.md, 중간 결과물)
│   │
│   │  [ingest 스캔 대상 — 위 2개 제외 전체, sub 폴더 재귀]
│   ├── clippings/             ← 웹 클리핑 (Obsidian Clipper)
│   │   └── mysql/             ← 분류용 sub 폴더 (자유 생성 가능)
│   ├── pdf/                   ← PDF 원본 (pdf2md 변환 후에도 여기 유지)
│   ├── (신규 분류 폴더)/      ← 분류·탐색 편의로 자유롭게 추가 가능
│   └── (기타 파일)            ← raw/ 루트에 직접 저장된 파일
└── wiki/
    ├── index.md               ← 전체 문서 지식 지도 (세션 시작 시 반드시 읽기)
    ├── log.md                 ← 변경 이력
    ├── DBMS/
    │   ├── PostgreSQL/
    │   ├── MySQL/
    │   ├── Oracle/
    │   ├── Altibase/
    │   ├── 쿼리튜닝/          ← DB 종류 무관, 최우선 분류
    │   ├── DW/
    │   └── (공통)/
    ├── ML(AI)/                ← ML·AI·딥러닝 (독립 최상위 카테고리)
    ├── kafka/                 ← Apache Kafka (독립 최상위)
    ├── elasticsearch/         ← Elasticsearch (독립 최상위)
    ├── airflow/               ← Apache Airflow (독립 최상위)
    ├── 프로그래밍/
    ├── 시스템/
    └── 기타(ETC)/             ← 위 카테고리에 속하지 않는 기타 문서
```

> `wiki/` 루트에는 `index.md`와 `log.md`만 존재합니다.
> 모든 지식 문서는 반드시 카테고리 하위 폴더에 저장합니다.

## 4. 특징 — Skills & Commands

token 사용량을 줄이고 한 번에 많은 문서를 처리할 때 발생하는 max request limit 제한을 우회하기 위해 아래의 Skill과 Command들이 사용된다.

### 글로벌 (`~/.claude/`)

| 구분 | 이름 | 트리거 | 기능 |
|------|------|--------|------|
| **skill** | `graphify` | `/graphify` | 지식 그래프 빌드·조회 |

### 프로젝트 (`D:\llmwiki\.claude/`)

| 구분 | 이름 | 트리거 | 기능 |
|------|------|--------|------|
| **skill** | `run-batch` | `/run-batch` | 대량 raw 원본 자동 배치 ingest (`--sleep <분>` 옵션) |
| **command** | `ingest` | `/ingest` | raw 원본 → wiki 문서 변환 |
| **command** | `lint` | `/lint` | wiki 품질 점검 → 보고서 생성 |

## 5. 핵심 참고 파일

본 문서를 참고하여 LLM Wiki를 생성하고자 한다면 다음 파일들이 핵심이다.

| 파일 | 설명 |
|------|------|
| `./CLAUDE.md` | 전체 LLM Wiki 구조 및 처리 프로세스, 주요 명령어 소개 |
| `./.schema/system_prompt.md` | 문서 작성에 대한 상세 지침 |
| `./.schema/template_note.md` | 문서 템플릿 |
