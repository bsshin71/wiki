## 1. 핵심 요약

중소규모 클러스터는 **마스터 노드 3대 고정 + 데이터 노드 증설** 모델로 split brain을 방지하며 확장합니다.
수십 TB 규모 로그성 데이터는 **Hot-Warm 아키텍처**(최근 데이터=SSD Hot존, 오래된 데이터=SATA Warm존)로 서버 비용·대수를 절감합니다.

---

## 2. 마스터/데이터 노드 분리 모델

- 대규모 클러스터에서 가장 많이 쓰는 모델.
- **마스터 노드 3대로 고정 설계**, 사용량 증가 시 **데이터 노드만 증설**.
- 장점:
  - 마스터 수가 늘지 않아 **split brain 방지**.
  - 용량 부족 시 데이터 노드만 추가 → 증설 용이.
- 중소규모 클러스터 대부분 이 모델로 구축.

## 3. Hot-Warm 아키텍처

> 수십 TB 규모가 되면 데이터 노드 급증 + SSD 비용 폭증 → Hot-Warm 도입.

| 존 | 디스크 | 용도 |
|----|--------|------|
| **Hot** | SSD | 조회 잦은 최근 데이터 |
| **Warm** | SATA | 오래되어 조회 뜸한 데이터 |

- 로그성 데이터: 최근 수 RB만 SSD(Hot), 나머지는 저렴한 SATA(Warm)로 이전.
- 효과: 대용량 클러스터에서도 **서버 대수·비용 대폭 절감**.
