## 1. 핵심 요약

On-premise Linux Kafka broker 튜닝 포인트는 **커널 파라미터(swappiness·dirty ratio·네트워크 버퍼)**, **디스크(xfs + noatime)**, **GC(G1 pause/IHOP)** 입니다.
모니터링은 System·Broker·Producer·Consumer Metric을 이해하고 구성 튜닝과 병행합니다.

---

## 2. 커널 파라미터 (sysctl)

```properties
vm.swappiness = 1                 # 스왑 최소화
vm.dirty_background_ratio = 5
vm.dirty_ratio = 60-80            # write 버퍼링(처리량↑)

# 네트워크 버퍼 (대용량 처리)
net.core.wmem_default=131072
net.core.rmem_default=131072
net.core.wmem_max=2097152
net.core.rmem_max=2097152
net.ipv4.tcp_wmem=65536 131072 2097152
net.ipv4.tcp_rmem=65536 131072 2097152
net.ipv4.tcp_window_scaling=1
net.ipv4.tcp_max_syn_backlog=8192
net.core.netdev_max_backlog=8192
```
> 클라우드 환경은 상이할 수 있음(on-premise 기준).

## 3. 디스크

- **ext4보다 xfs** 권장.
- mount option **`noatime`** — access time write 동작을 줄여 I/O 절감.

## 4. Garbage Collection (G1)

| 옵션 | 의미 |
|------|------|
| `MaxGCPauseMillis` | 목표 GC pause 시간 |
| `InitiatingHeapOccupancyPercent` (IHOP) | concurrent GC 시작 힙 점유율 |
