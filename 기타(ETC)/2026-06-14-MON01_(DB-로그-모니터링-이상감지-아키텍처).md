## 1. 핵심 요약

30대 이상 DB/ProxySQL 서버의 에러·슬로우·ProxySQL 로그를 **자동 수집→ES 저장→이상 감지→알림**하는 통합 시스템. 1차는 filebeat→Logstash→ES + ElastAlert 구성으로 시작하고, 2차에 Kafka 버퍼를 추가한다.

---

## 2. 아키텍처 단계

### 1차 구현 (현행)
```
DB/ProxySQL 서버
  └─ filebeat(로그 수집)
      └─ Logstash(변환·파싱)
          └─ Elasticsearch(저장)
              ├─ ElastAlert(rule 기반 이상 감지 → Slack/webhook 알림)
              └─ Kibana(대시보드)
              └─ Airflow DAG(배치·통계성 이상 감지)
```

### 2차 구현 (예정)
- Kafka 버퍼 추가: filebeat → **Kafka** → Confluent ES Sink Connector → ES

## 3. 구성 요소

| 구성 요소 | 역할 |
|-----------|------|
| filebeat | 각 DB 서버 로그 수집 (mysql-err·slow·ProxySQL) |
| Logstash | msg_type 필드로 분류·파싱 후 ES 적재 |
| Elasticsearch | `swz-myerr-*`·`swz-myslow-*`·`swz-proxylog-*` 인덱스 |
| ElastAlert | 간단한 rule 기반 이상 감지 (ERROR 발생, 빈도 초과 등) |
| Airflow | 배치·통계성 감지 (예: ES 쿼리로 백업 완료 여부 확인) |
| 알람발송서버 | HTTP POST 수신 후 Slack·Telegram·SMS 등 발송 |

## 4. 설계 원칙

- **Docker 최대 활용**: 설치·구축 리소스 최소화
- ElastAlert: 단순 rule (1분 내 ERROR 1건 이상 등)
- 복잡한 통계성 감지: Airflow DAG로 처리
