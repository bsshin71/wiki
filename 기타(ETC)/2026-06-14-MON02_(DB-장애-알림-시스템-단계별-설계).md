## 1. 핵심 요약

DB 장애 알림 시스템을 **1차(PMM Grafana Alert·단기)→2차(AlertManager 통합)→3차(자체 webhook receiver·최종)** 3단계로 발전시킨다. 1차는 단기간에 구성 가능하고, 3차는 PMM 외 이벤트·다채널 알림·설정 관리까지 지원한다.

---

## 2. 단계별 설계

### 1차 목표 (PMM Grafana AlertRule — 단기)
- PMM Grafana의 **AlertRule** 기능 사용
- host별·metric별 alert 설정 → Slack/email 발송
- **별도 구현 없이 Grafana 메뉴**로 빠르게 구성
- 2차·3차 완료 전까지 한시적 운용

### 2차 목표 (AlertManager 통합)
- 더 많은 기능을 제공하는 **Prometheus AlertManager** 사용
- route 트리로 alert 분류, receiver 그룹별 발송, inhibit_rules로 중복 억제

### 3차 목표 (최종 — 자체 webhook receiver)
- **PMM 외 이벤트도 알림** 가능 (DB-ELK·애플리케이션 등)
- 구현 모듈:
  - `webhook receiver`: alert 이벤트 수신 웹서버 (Python/Go/Node.js)
  - `alert sender`: alert DB 폴링 → 신규 이벤트 전송
  - `alert config manager`: 수신자·그룹·채널·반복주기 설정
