## 1. 핵심 요약
- CentOS/RHEL에서 `ifcfg` 파일 생성으로 고정 IP 설정. NetworkManager가 DHCP를 우선할 수 있으므로 `nmcli`로 강제 적용 필요.

## 2. 상세 설명

### 기본 설정 (ifcfg 파일 생성)

```bash
sudo vi /etc/sysconfig/network-scripts/ifcfg-enp0s8
```

**설정 파일 내용** (권장 — NetworkManager 간섭 방지):
```ini
TYPE=Ethernet
PROXY_METHOD=none
BROWSER_ONLY=no
BOOTPROTO=static       # dhcp 아닌 static (필수)
DEFROUTE=yes
IPV4_FAILURE_FATAL=no
NAME=enp0s8
DEVICE=enp0s8
ONBOOT=yes
IPADDR=192.168.138.104
NETMASK=255.255.255.0
GATEWAY=192.168.138.1
DNS1=1.1.1.1
PEERDNS=no             # DHCP 서버의 DNS 정보 무시
```

---

### 설정 적용

```bash
# 방법 1: 네트워크 서비스 재시작
sudo systemctl restart network

# 방법 2: nmcli로 강제 적용 (권장 — DHCP 덮어쓰기 방지)
sudo nmcli connection reload
sudo nmcli connection down enp0s8
sudo nmcli connection up enp0s8
```

---

### IP 확인

```bash
ip addr show enp0s8
# inet 192.168.138.104/24 ... 가 보이면 성공
```

---

### 트러블슈팅: DHCP가 계속 IP를 덮어쓸 때

```bash
# DHCP 임대 기록 삭제
sudo dhclient -r enp0s8
sudo systemctl restart network
```

**체크 포인트:**
1. `BOOTPROTO=static` (dhcp 아닌지 확인)
2. `PEERDNS=no` 누락 여부
3. VirtualBox 사용 시: 어댑터 2가 **호스트 전용(Host-Only)** 또는 **브리지** 로 설정되어 있는지 확인

---

### VirtualBox 환경 주의사항

- `enp0s3` (어댑터 1): NAT
- `enp0s8` (어댑터 2): 호스트 전용 어댑터(Host-Only) 또는 브리지 어댑터
- VirtualBox 메인 창 → 설정 → 네트워크 → 어댑터 2 → "네트워크 어댑터 활성화" 체크 확인
