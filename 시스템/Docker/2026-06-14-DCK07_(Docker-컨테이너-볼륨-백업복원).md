## 1. 핵심 요약

Docker 백업은 **컨테이너 환경(이미지)** 과 **데이터(볼륨)** 를 나눠 생각해야 한다. 이미지는 `commit+save/load`, 볼륨은 `tar` 또는 `--volumes-from`으로 백업한다.

---

## 2. 방법별 정리

### 방법 1 — 이미지로 백업 (환경 보존)
```bash
docker commit <container_id> backup-image
docker save -o backup.tar backup-image
# 복원
docker load -i backup.tar
docker run -d backup-image
```

### 방법 2 — 볼륨 백업 (데이터 보존)
```bash
# --volumes-from 방식
docker run --rm --volumes-from <container_id> \
  -v $(pwd):/backup ubuntu \
  tar cvf /backup/backup.tar /data
# 복원
docker run --rm --volumes-from <new_container_id> \
  -v $(pwd):/backup ubuntu \
  tar xvf /backup/backup.tar -C /
```
> ⚠️ 컨테이너 실행 중 볼륨 백업 시 데이터 손실 가능 → **중지 후 백업** 권장.

### 방법 3 — Docker Compose 백업
```bash
tar czvf compose_backup.tar.gz docker-compose.yml .env
# 복원
tar xzvf compose_backup.tar.gz && docker compose up -d
```

## 3. 어떤 방법을 쓸까?

| 상황 | 방법 |
|------|------|
| 컨테이너 환경 유지 | `commit + save` |
| 데이터(볼륨)만 유지 | `tar` 볼륨 백업 |
| Compose 프로젝트 | `docker-compose.yml + .env` 백업 |
