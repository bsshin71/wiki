## 1. 핵심 요약

Docker 볼륨 데이터를 영구 보존하는 4가지 백업 방법: ① 호스트 경로 직접 tar 압축(가장 간단), ② `docker run --volumes-from`으로 컨테이너 내부 백업, ③ `docker cp`로 파일 직접 복사, ④ `docker volume` 활용. 대부분 ①이 제일 빠르고 간편하다.

---

## 2. 방법 1 — 호스트 경로 직접 tar (권장)

```bash
# 백업 (예: pmm_pmm-data 볼륨)
tar -czvf pmm_pmm-data-backup.tar.gz \
  -C /var/lib/docker/volumes/pmm_pmm-data/_data .

# 복원
tar -xzvf pmm_pmm-data-backup.tar.gz \
  -C /var/lib/docker/volumes/pmm_pmm-data/_data
```

## 3. 방법 2 — docker run + --volumes-from

```bash
# 백업 (컨테이너 내부 /srv 기준)
docker run --rm --volumes-from <컨테이너_ID> \
  -v $(pwd):/backup ubuntu \
  tar czvf /backup/pmm_pmm-data-backup.tar.gz -C /srv .

# 복원
docker run --rm --volumes-from <컨테이너_ID> \
  -v $(pwd):/backup ubuntu \
  tar xzvf /backup/pmm_pmm-data-backup.tar.gz -C /srv
```

## 4. 방법 3 — docker cp

```bash
# 백업
docker cp <컨테이너_ID>:/srv ./pmm_pmm-data-backup
tar -czvf pmm_pmm-data-backup.tar.gz pmm_pmm-data-backup

# 복원
tar -xzvf pmm_pmm-data-backup.tar.gz
docker cp pmm_pmm-data-backup <컨테이너_ID>:/srv
```

## 5. 방법 4 — docker volume 직접

```bash
# 백업
docker run --rm \
  -v pmm_pmm-data:/data \
  -v $(pwd):/backup ubuntu \
  tar czvf /backup/pmm_pmm-data-backup.tar.gz -C /data .

# 복원
docker run --rm \
  -v pmm_pmm-data:/data \
  -v $(pwd):/backup ubuntu \
  tar xzvf /backup/pmm_pmm-data-backup.tar.gz -C /data
```
